## v1 Requirements

### Authentication

- [x] **AUTH-01**: User can create account with email and password
- [x] **AUTH-02**: User can log in via Google OAuth
- [x] **AUTH-03**: User can log in via LinkedIn OAuth (profile data imported as secondary source)
- [x] **AUTH-04**: User session persists across browser refresh
- [x] **AUTH-05**: User can log out from any page
- [x] **AUTH-06**: User can reset password via email link

### Onboarding

- [x] **ONBD-01**: User can upload CV (PDF, DOCX, TXT) stored in S3
- [x] **ONBD-02**: CV is parsed via text extraction + LLM to produce structured profile (roles, skills, qualifications, sectors, experience years)
- [x] **ONBD-03**: User sees draft profile as editable cards and can confirm/adjust
- [x] **ONBD-04**: User sets availability (full-time, part-time, occasional, project-only)
- [x] **ONBD-05**: User sets preferences (domains of interest, willing to mentor/volunteer, max Circles, communication preference)
- [x] **ONBD-06**: Stripe Connect onboarding triggered if user opts into paid work
- [x] **ONBD-07**: First wellbeing check-in completed during onboarding
- [x] **ONBD-08**: CV upload to live profile completes in under 5 minutes

### Challenges

- [x] **CHAL-01**: User can browse challenge feed sorted by relevance, recency, domain, or deadline
- [x] **CHAL-02**: Challenge cards show title, brief excerpt, domain, skills, type, deadline, interest count
- [x] **CHAL-03**: User can filter challenges by domain, skill match, type (free/paid), and timeline
- [x] **CHAL-04**: User sees top 3-5 "recommended for you" challenges based on profile match
- [x] **CHAL-05**: User can express interest in a challenge with single tap
- [x] **CHAL-06**: Community manager can create and post challenges via admin interface

### Matching

- [x] **MTCH-01**: Platform scores contributors against challenges (direct skill match, cross-functional value, availability, Circle diversity, active Circle count)
- [x] **MTCH-02**: Community manager reviews suggested team and confirms or adjusts Circle composition

### Circles

- [x] **CIRC-01**: Circle is formed with 3-7 members linked to a challenge
- [x] **CIRC-02**: Circle workspace shows pinned challenge brief, notes feed, member list, and social channel link
- [x] **CIRC-03**: Circle members can post notes with text and file attachments
- [x] **CIRC-04**: Circle members can set/change external social channel (WhatsApp/Slack/Discord/Teams/Signal)
- [x] **CIRC-05**: Circle can submit structured resolution (problem summary, recommendations, evidence, dissenting views, implementation notes)
- [x] **CIRC-06**: Challenger can rate resolution (1-5) and provide feedback
- [x] **CIRC-07**: Contributors can participate in multiple Circles simultaneously (configurable max, default 3)
- [x] **CIRC-08**: New members can join a Circle mid-challenge

### Payments

- [x] **PAY-01**: Contributor can onboard to Stripe Connect (bank details for payouts)
- [x] **PAY-02**: Knowledge Transition retainer: monthly Stripe subscription with auto 75/25 split
- [x] **PAY-03**: Premium challenge stipend: fixed fee held via payment intent, released on resolution (75/25 split)
- [x] **PAY-04**: SME subscription: monthly/annual Stripe subscription for ongoing expertise pool access
- [x] **PAY-05**: Contributors receive payouts on Stripe's standard schedule

### Wellbeing

- [x] **WELL-01**: User completes wellbeing check-in (UCLA Loneliness Scale short form + WEMWBS) at onboarding
- [x] **WELL-02**: Wellbeing check-in prompted every 8 weeks
- [x] **WELL-03**: Wellbeing data stored with GDPR special category protections (explicit consent, DPIA)

### Impact

- [x] **IMPT-01**: Contributor sees challenges participated in (active + historical)
- [x] **IMPT-02**: Contributor sees total hours contributed (self-reported per Circle)
- [x] **IMPT-03**: Contributor sees total earnings (if opted into paid work)
- [x] **IMPT-04**: Contributor sees unpaid contribution formally recognised (hours mentoring, volunteering, community work)
- [x] **IMPT-05**: Contributor sees wellbeing trajectory (personal scores over time)
- [x] **IMPT-06**: Challenger sees recommendations received, implementation status, and feedback submitted

### Notifications

- [x] **NOTF-01**: User notified of new challenge match
- [x] **NOTF-02**: User notified when Circle is formed
- [x] **NOTF-03**: User notified of Circle activity (configurable: immediate, daily digest, off)
- [x] **NOTF-04**: User notified when wellbeing check-in is due
- [x] **NOTF-05**: User notified of resolution feedback
- [x] **NOTF-06**: User notified of payment received
- [x] **NOTF-07**: Notifications delivered via PWA push (primary) and email fallback

### Platform

- [x] **PLAT-01**: MCP server exposes 14 tools across 4 domains (Contributors, Challenges, Circles, Wellbeing)
- [x] **PLAT-02**: Basic React/Vite web UI for all contributor-facing flows
- [x] **PLAT-03**: PWA manifest, service worker, and home screen install
- [x] **PLAT-04**: WCAG AAA contrast (7:1 minimum) with 16px+ body text
- [x] **PLAT-05**: External social deep links launch correctly (WhatsApp, Slack, Discord, Teams, Signal)
- [x] **PLAT-06**: GDPR-compliant data handling (consent management, data export, deletion)
- [x] **PLAT-07**: Community manager admin interface for challenge posting and Circle management
