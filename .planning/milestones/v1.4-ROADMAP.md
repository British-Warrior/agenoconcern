# Milestone v1.4: WCAG Compliance

**Status:** SHIPPED 2026-04-16
**Phases:** 19-23
**Total Plans:** 10

## Overview

Full WCAG 2.2 AA audit and remediation across the entire application — keyboard navigation, screen reader support, focus management, form accessibility, motion respect, semantic HTML, ARIA patterns, and a CI regression gate that prevents new violations reaching production.

## Phases

### Phase 19: Foundation
**Goal**: Global SPA accessibility baseline — route change announcements, lint-time violation detection, lang/motion/aria-current trivial wins.
**Depends on**: Phase 18
**Requirements**: KBD-03, VIS-03, VIS-04, TOOL-01, TOOL-04
**Plans**: 2 plans

Plans:
- [x] 19-01: RouteChangeSync component, aria-current on Navbar links, prefers-reduced-motion CSS, lang verification
- [x] 19-02: eslint-plugin-jsx-a11y strict config, fix all existing violations

**Details:**
- RouteChangeSync mounted in all 4 layout shells (AppShell + 3 guard routes)
- NavLink `end` prop on all nav items prevents false-positive active states
- ESLint flat config (eslint.config.js) with jsx-a11y/strict preset
- 18 pre-existing a11y violations fixed across 8 files

### Phase 20: Focus Infrastructure
**Goal**: Shared accessibility primitives — focus trap, modal wrapper, live region announcer, and corrected focus ring contrast.
**Depends on**: Phase 19
**Requirements**: KBD-01, KBD-02, VIS-01, TOOL-03
**Plans**: 2 plans

Plans:
- [x] 20-01: useFocusTrap hook, Modal shared wrapper, AnnounceProvider and useAnnounce hook
- [x] 20-02: Focus ring contrast — replace all semi-transparent and non-compliant focus rings with focus-visible:ring-accent-500

**Details:**
- useFocusTrap: custom hook, 6 focusable selectors, re-query on each keydown
- Modal: createPortal to document.body, role=dialog, Escape to close
- AnnounceProvider: clear-then-set (50ms) for duplicate re-announcement
- accent-500 token standardised across all 20 files

### Phase 21: Interactive Component Remediation

**Depends on**: Phase 20
**Requirements**: KBD-04, KBD-05, SR-01
**Plans**: 2 plans

Plans:
- [x] 21-01: Migrate CircleFormationModal, AddMemberModal, ConfirmDialog to shared Modal wrapper; add aria-expanded to all disclosure widgets
- [x] 21-02: NotificationBell keyboard operability — keyboard open/close, item navigation, aria-expanded

**Details:**

- Toggle button placed after chart container for logical tab order

## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**
- RouteChangeSync mounted in all 4 layout shells for complete route coverage
- ESLint flat config (v9+) with jsx-a11y/strict preset (strictest available)
- Custom useFocusTrap hook over external library — 6 focusable selectors, re-query on keydown
- Modal: createPortal to document.body with clear-then-set announcement pattern
- focus-visible prefix used throughout — rings on keyboard only, not mouse click
- accent-500 token standardised across all files for focus ring consistency
- role=menu/menuitem for NotificationBell (actions, not selections)
- best-practice tag in axe fixture captures heading-order rules not in wcag2aa
- Chromium-only CI sufficient for axe-core rule coverage

**Issues Resolved:**
- 18 pre-existing jsx-a11y violations fixed across 8 files
- PortalDashboard h1 was outside main landmark — fixed for RouteChangeSync
- Rate limiter IPv6 validation error in api-key-auth middleware
- overflow-hidden clipping focus rings on ChallengeRow cards
- Missing focus-within outlines on WellbeingForm radio labels

**Issues Deferred:**
- NVDA + VoiceOver cross-browser compatibility matrix (v2+ A11Y-01)
- Windows High Contrast Mode smoke test (v2+ A11Y-02)
- recharts accessibilityLayer evaluation (v2+ A11Y-03)

**Technical Debt Incurred:**
- axe-core covers ~57% of WCAG violations — remaining coverage requires ongoing manual testing
- Authenticated page axe scans not yet automated (require test auth setup)
