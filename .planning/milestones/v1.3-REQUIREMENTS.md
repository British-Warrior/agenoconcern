## v1.3 Requirements

Requirements for v1.3 Enhanced Reporting & Institution Portal. Each maps to roadmap phases.

### Wellbeing Reporting

- [x] **WELL-01**: PDF impact report includes an anonymous wellbeing band (low/typical/high) when the institution has enough opted-in contributors
- [x] **WELL-02**: Wellbeing band is derived from Rasch-transformed SWEMWBS metric scores, not raw sums
- [x] **WELL-03**: Wellbeing data is suppressed (band hidden) when fewer than k contributors have completed a check-in at the institution
- [x] **WELL-04**: Only contributors who have consented to the institutional_reporting purpose are included in wellbeing aggregation

### Attention Analytics

- [x] **ATTN-05**: CM can see a trend direction indicator (increasing/stable/decreasing) for attention flags per institution on the dashboard
- [x] **ATTN-06**: CM can view a weekly trend chart of attention flag counts per institution over time

### Scheduled Delivery

- [x] **SCHED-01**: CM can enable or disable automatic PDF report delivery per institution
- [x] **SCHED-02**: When enabled, the PDF impact report is automatically emailed to the institution's contact email on the configured schedule (weekly or monthly)
- [x] **SCHED-03**: Each delivery attempt is logged with timestamp, status (sent/failed), and recipient email
- [x] **SCHED-04**: Failed deliveries are automatically retried with exponential backoff

### Institution Portal

- [x] **PORTAL-01**: Institution contacts can log in with their own credentials (separate from contributor auth)
- [x] **PORTAL-02**: Institution portal shows a read-only dashboard with their institution's stats (contributor count, challenges, hours)
- [x] **PORTAL-03**: Institution users can download their own PDF impact report without CM intervention
- [x] **PORTAL-04**: Institution users can view attention flags at their institution (read-only, no resolve action)
