## v1.1 Requirements

### UX Overhaul

- [x] **UX-01**: User can navigate to dashboard, challenges, circles, impact, and logout from every authenticated page
- [x] **UX-02**: Dashboard shows active circles, open challenge matches, earnings summary, hours contributed, and wellbeing check-in status
- [x] **UX-03**: Wellbeing scores display with context (score/max, colour band: low/average/high per UK norms, trend direction)
- [x] **UX-04**: Wellbeing trajectory shows as a line chart with SWEMWBS and UCLA series over time
- [x] **UX-05**: All interactive elements (cards, links, buttons) have visible hover states, cursor pointer, and click affordance
- [x] **UX-06**: CM can view and access all circles they formed from the circles page
- [x] **UX-07**: CM circles page shows role-appropriate messaging and actions (not "Express interest in challenges")
- [x] **UX-08**: User can reach wellbeing check-in from navigation without typing the URL
- [x] **UX-09**: Error messages display human-readable text (no raw UUIDs or technical details)
- [x] **UX-10**: Edit Resolution button has clear active/disabled states and works when clickable
- [x] **UX-11**: Circle formation modal shows correct member names matching selected contributors

### Wellbeing Visualisation

- [x] **WELL-01**: Impact dashboard shows wellbeing trajectory as an interactive line chart (recharts)
- [x] **WELL-02**: Wellbeing chart includes UK population benchmark line for SWEMWBS
- [x] **WELL-03**: Wellbeing scores include accessible data table alternative for screen readers

### VANTAGE Integration

- [x] **VANT-01**: Server supports API key authentication via X-API-Key header, separate from JWT cookie auth
- [x] **VANT-02**: API keys are stored as SHA-256 hashes with scoped permissions and expiry
- [x] **VANT-03**: API key endpoints are rate-limited (express-rate-limit)
- [x] **VANT-04**: VANTAGE endpoint contract is documented (request/response shapes for all REST endpoints)

### Kiosk Mode

- [x] **KIOSK-01**: URL parameter `?kiosk=true` activates kiosk mode with simplified UI (larger buttons, no install/notification prompts)
- [x] **KIOSK-02**: Kiosk sessions auto-logout after 10 minutes of inactivity with 60-second warning countdown
- [x] **KIOSK-03**: Kiosk logout clears server cookies AND React Query cache completely (no data leakage)
- [x] **KIOSK-04**: Kiosk mode shows "End Session" button prominently in navigation

### Challenger Portal

- [x] **CHAL-01**: Organisation can register an account with name, email, and organisation type
- [x] **CHAL-02**: Organisation can submit a challenge brief (title, description, domain, skills, type, deadline, circle size)
- [x] **CHAL-03**: Submitted challenges enter "draft" status for CM review before publishing
- [x] **CHAL-04**: Organisation can view status of their submitted challenges (draft, open, closed, archived)
- [x] **CHAL-05**: Organisation can view circle progress on their challenges (members, status)
- [x] **CHAL-06**: Organisation can rate completed resolutions and provide feedback

### Institutional Landing Pages

- [x] **INST-01**: Each institution has a public landing page at `/i/[slug]` with name, description, and "Get Started" CTA
- [x] **INST-02**: Institutional landing page shows aggregate local impact stats (contributors, challenges, hours)
- [x] **INST-03**: Institutional URL auto-activates kiosk mode when accessed from that page
