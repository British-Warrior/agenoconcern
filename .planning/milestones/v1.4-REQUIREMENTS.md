## v1.4 Requirements

Requirements for v1.4 WCAG Compliance. Each maps to roadmap phases.

### Keyboard & Focus

- [x] **KBD-01**: All modals trap focus within the dialog and return focus to the trigger on close
- [x] **KBD-02**: Escape key closes all modals and dropdowns
- [x] **KBD-03**: SPA route changes move focus to the main content heading
- [x] **KBD-04**: NotificationBell dropdown is keyboard-operable with aria-expanded
- [x] **KBD-05**: All notification items are reachable and activatable via keyboard

### Screen Reader

- [x] **SR-01**: All disclosure widgets have aria-expanded state
- [x] **SR-02**: Form errors are linked to inputs via aria-describedby
- [x] **SR-03**: File drop zones have accessible labels
- [x] **SR-04**: Badge count changes are announced to screen readers
- [x] **SR-05**: Loading skeletons have role="status" and aria-busy
- [x] **SR-06**: AttentionTrendChart has a companion data table for screen readers

### Visual & Layout

- [x] **VIS-01**: Focus indicators meet 3:1 contrast ratio (WCAG 2.4.11)
- [x] **VIS-02**: Heading hierarchy is correct across all pages
- [x] **VIS-03**: `<html lang="en">` is set
- [x] **VIS-04**: Animations respect prefers-reduced-motion

### Forms & Auth

- [x] **FORM-01**: Auth form inputs have autocomplete attributes (WCAG 1.3.5)
- [x] **FORM-02**: Password inputs allow paste (WCAG 3.3.8)

### Tooling & Regression

- [x] **TOOL-01**: eslint-plugin-jsx-a11y in strict mode catches static violations
- [x] **TOOL-02**: axe-core Playwright tests run in CI on every PR
- [x] **TOOL-03**: Centralised announce() utility provides live region feedback for async operations
- [x] **TOOL-04**: Navigation links use aria-current="page"

## Out of Scope

| Feature | Reason |
|---------|--------|
| Accessibility overlay widget | Doesn't fix underlying issues; creates false compliance impression |
| AAA compliance beyond colour contrast | AA is the legal standard; AAA colour already met via OKLCH |
| Custom screen reader testing framework | Manual testing with NVDA + Firefox is sufficient for pilot |
| ARIA on semantic HTML elements | Redundant ARIA causes screen reader confusion |
| Positive tabindex values | Breaks natural tab order; impossible to maintain |
