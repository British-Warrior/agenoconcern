# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-03-15
**Phases:** 1-6
**Total Plans:** 19

## Overview

Indomitable Unity v1.0 MVP delivers a pilot-ready expertise marketplace that takes experienced professionals from CV upload to paid advisory work and community contribution through Circles. The build progressed from foundation and auth, through the core onboarding and challenge loop, into Circles collaboration, payments, wellbeing and impact tracking, and finally notifications and PWA polish.

## Milestone Summary

**Key Decisions:**
- MCP server architecture for VANTAGE integration (tool-based API, future-proof)
- Basic web UI for MVP (React/Vite) — VANTAGE overlays later
- External social over internal chat (WhatsApp/Slack/Discord deep links only)
- 75/25 Stripe Connect payment splits (contributor-favourable)
- Semi-manual challenge intake via community manager (self-service is v2)
- CV upload as onboarding (zero form-filling, emotionally reframes expertise)
- Match scoring in TypeScript (not SQL) to avoid Drizzle JSONB bug
- Shared package consumed as TS source (no build step)
- Tailwind v4 CSS-first config with OKLCH palette for WCAG AAA

**Issues Resolved:**
- Notes pagination contract mismatch (Phase 4) — fixed hasMore → nextCursor
- Hours logging wrong API path (Phase 5) — fixed /api/impact/hours → /api/payments/hours
- PWA navigateFallback missing (Phase 6) — fixed for offline SPA deep-link navigation
- Stripe Connect integration and wellbeing trajectory display — fixed post-audit

**Issues Deferred:**
- Employment Agencies Act 1973 classification — needs legal advice before payments go live
- NOTF-05 resolution_feedback trigger — infrastructure ready, trigger deferred
- DPIA and Appropriate Policy Document — legal documents external to codebase (user-approved)
- WEMWBS licence registration with Warwick Medical School (user-approved)

**Technical Debt Incurred:**
- No formal VERIFICATION.md for Phase 1 (assessed via plan summaries)
- ChallengeForm.tsx TODO for payments field (deferred by design)
- CM challenges use client-side filtering (limit 100) — adequate at pilot scale
