# Milestone v1.3: Enhanced Reporting & Institution Portal

**Status:** SHIPPED 2026-03-30
**Phases:** 16-18
**Total Plans:** 6

## Overview

CM sees richer analytics in the dashboard and PDF; reports deliver themselves on a schedule; institution contacts have their own read-only portal.

## Phases

### Phase 16: Wellbeing & Attention Analytics

**Goal:** The CM dashboard and PDF impact report surface anonymised wellbeing bands and attention trend data derived from real contributor metrics.
**Depends on:** Phase 15 (PDF infrastructure, attention dashboard)
**Requirements:** WELL-01, WELL-02, WELL-03, WELL-04, ATTN-05, ATTN-06
**Plans:** 2 plans

Plans:
- [x] 16-01-PLAN.md — Wellbeing aggregation service: DB migration, Rasch utility, consent checkbox, wellbeing band query, PDF extension
- [x] 16-02-PLAN.md — Attention trend API endpoint, trend chart component, dashboard integration with indicator and Trends tab

**Completed:** 2026-03-27

### Phase 17: Scheduled Report Delivery

**Goal:** PDF impact reports are emailed automatically on a CM-configured schedule, with every attempt logged and failures retried.
**Depends on:** Phase 16 (complete PDF content)
**Requirements:** SCHED-01, SCHED-02, SCHED-03, SCHED-04
**Plans:** 2 plans

Plans:
- [x] 17-01-PLAN.md — Schema migration, cron job with advisory lock, PDF email delivery, CM toggle UI with cadence selector
- [x] 17-02-PLAN.md — Delivery log viewer UI, end-to-end verification checkpoint

**Completed:** 2026-03-30

### Phase 18: Institution Portal

**Goal:** Institution contacts can log in with their own credentials and self-serve their institution's stats, PDF, and attention flags — without CM involvement.
**Depends on:** Phase 17 (scheduled delivery makes portal download independently useful)
**Requirements:** PORTAL-01, PORTAL-02, PORTAL-03, PORTAL-04
**Plans:** 2 plans

Plans:
- [x] 18-01-PLAN.md — Portal auth: DB table, JWT middleware, server routes (login/refresh/me), frontend auth context + route guard + login page
- [x] 18-02-PLAN.md — Portal dashboard (stats, PDF download, attention flags read-only), CM account creation UI

**Completed:** 2026-03-30

## Milestone Summary

**Key Decisions:**
- Phase 16: k=5 anonymity threshold (ONS guidance, pilot-scale viable)
- Phase 16: Rasch-transformed SWEMWBS scores, not raw sums, for band classification
- Phase 16: Modal band with Typical tie-break for wellbeing aggregation
- Phase 16: Consent checkbox at wellbeing check-in (not grandfathered)
- Phase 17: Advisory lock key 7171 for report delivery job (pg_try_advisory_lock)
- Phase 17: Exponential backoff: 15min base * 2^(attempt-1) + jitter, max 4h, 5 attempts
- Phase 18: Separate Express router at /api/portal (not subdomain)
- Phase 18: Portal JWT type=portal discriminant for auth isolation
- Phase 18: Partial unique index enforces one active portal account per institution

**Issues Resolved:**
- GET /admin/institutions was missing new delivery columns (contactEmail, schedule fields) — fixed during 17-02 checkpoint

**Issues Deferred:**
- SWEMWBS commercial licence confirmation before wellbeing band goes live to institutions
- Employment Agencies Act 1973 classification (carried from v1.1)

**Technical Debt Incurred:**
- None significant
