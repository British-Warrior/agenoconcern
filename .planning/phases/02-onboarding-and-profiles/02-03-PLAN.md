      - **Communication Channel**: radio group -- Email, Phone
      - **Communication Frequency**: radio group -- Immediate, Daily digest, Weekly digest
    - "Save Preferences" button calls useSavePreferences mutation
    - On success: navigate to /onboarding/stripe

    - Use existing design system components. Form layout: stacked sections with clear headings.
    - WCAG AAA: all inputs in fieldsets with legends, radio groups properly grouped, 48px touch targets on checkboxes/radios

    - Choice-focused framing per locked decision:
      - Heading: "Want to earn from your expertise?"

    - Two prominent buttons:
      - "Set up payments" (primary) -> calls useStartStripeConnect, which redirects to Stripe hosted onboarding
      - "Set up later" (secondary/outline, equally visible) -> calls useSkipStripe, on success navigates to /onboarding/complete
    - Note below: "You can always set this up later from your profile"
    - If arriving from Stripe return (check URL for stripe-return or similar indicator from backend redirect): show status message ("Payment setup complete!" or "Payment setup incomplete -- you can finish later")

    - Summary: "Your profile is live and you're ready to explore challenges"
    - "Go to Dashboard" button navigates to /dashboard
    - Auto-redirect to /dashboard after 5 seconds (with visible countdown or progress bar)
    - This page handles both Stripe-complete and Stripe-skipped flows
