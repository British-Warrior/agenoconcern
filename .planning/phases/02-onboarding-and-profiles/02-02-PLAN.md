  <action>
    **Install dependency** (from packages/web):
    ```
    npm install react-dropzone
    ```

    **1. API client** (`packages/web/src/api/onboarding.ts`):
    - Use the existing fetch wrapper from `api/client.ts` (auto 401 refresh pattern)
    - Export `onboardingApi` object with methods:
      - `getUploadUrl(fileName, mimeType)` -> POST /api/onboarding/upload-url -> { uploadUrl, s3Key }
      - `uploadToS3(uploadUrl, file)` -> PUT directly to S3 presigned URL with Content-Type header
      - `uploadImage(file)` -> POST /api/onboarding/upload-image (FormData with file field)
      - `startParse(params: { s3Key, mimeType } | { extractedText })` -> POST /api/onboarding/start-parse -> { jobId }
      - `getParseStatus(jobId)` -> GET /api/onboarding/parse-status/:jobId -> { id, status, errorMessage, profile? }
      - `getProfile()` -> GET /api/onboarding/profile
      - `updateProfile(data)` -> PUT /api/onboarding/profile
      - `savePreferences(data)` -> PUT /api/onboarding/preferences
      - `startStripeConnect()` -> POST /api/onboarding/stripe/connect -> { url }
      - `skipStripe()` -> POST /api/onboarding/stripe/skip

    **2. Onboarding hooks** (`packages/web/src/hooks/useOnboarding.ts`):
    - `useCvUpload()` -> useMutation that: (a) gets presigned URL or uploads image based on file type, (b) for electronic files: fetches presigned URL, PUTs file to S3, calls startParse with s3Key+mimeType, (c) for images (jpg/png): calls uploadImage, then startParse with extractedText. Returns { mutate, jobId, isPending, error }.
    - `useCvParseStatus(jobId: string | null)` -> useQuery with refetchInterval: polls every 2s while status is "pending" or "processing", stops on "complete" or "failed". Enabled only when jobId is truthy.
    - `useProfile()` -> useQuery for GET /api/onboarding/profile
    - `useUpdateProfile()` -> useMutation for PUT /api/onboarding/profile, invalidates profile query on success

    **3. Upload CV page** (`packages/web/src/pages/onboarding/UploadCV.tsx`):
    - Use react-dropzone's `useDropzone` hook
    - Accept types: PDF, DOCX, TXT, JPG, JPEG, PNG. Max size 10MB. Max 1 file.
    - Layout: centered card with large drop zone area (dashed border), drag-active state changes border colour
    - Inside drop zone: icon, "Drag your CV here" heading, "or click to browse" subtext, accepted formats listed
    - On file drop/select: show file name + size, start upload via useCvUpload mutation
    - Upload progress: show "Uploading..." state with the existing Button loading pattern
    - On successful upload (jobId returned): navigate to /onboarding/parsing?jobId={jobId}
    - Error state: show error in Alert component below the drop zone
    - Use existing design system components (Card, Button, Alert) and Tailwind classes
    - Follow WCAG AAA: 48px touch targets, clear focus states, screen reader labels on the dropzone

    **4. Parsing wait page** (`packages/web/src/pages/onboarding/Parsing.tsx`):
    - Read jobId from URL search params
    - Use useCvParseStatus(jobId) to poll
    - Animated step-by-step progress labels (Claude's discretion on exact approach):
      - Show sequential steps that reveal over time: "Reading your CV...", "Finding your expertise...", "Building your profile...", "Almost there..."
      - Use CSS transitions/animations (Tailwind animate classes) for smooth appearance
      - Steps appear every ~3 seconds regardless of actual progress (the polling completes when it completes)
    - On status "complete": navigate to /onboarding/review
    - On status "failed": show error message with "Try again" button linking back to /onboarding/upload
    - No frozen spinner -- this must feel alive and intentional
