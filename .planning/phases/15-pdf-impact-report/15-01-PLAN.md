**A. Create `packages/server/src/pdf/institution-report.ts`:**

Use the ESM-safe pattern from research — `fileURLToPath(import.meta.url)` + `dirname` to resolve font paths. Never use pdfkit built-in font names.

Export interface `ReportData`:
```typescript
export interface ReportData {
  institutionName: string;
  institutionCity: string | null;
  stats: { contributors: number; challenges: number; hours: number };
  generatedAt: Date;
  dateRange: { startDate: Date | null; endDate: Date | null };
}
```

Export function `buildInstitutionReport(data: ReportData): PDFKit.PDFDocument`:
- Create `new PDFDocument({ font: FONTS.regular, size: "A4", margin: 50, info: { Title, Author: "Indomitable Unity" } })`
- Header band: navy (#1a1d2e) rect across full width (0,0 to page.width, 80)
  - "Indomitable Unity" in white, bold, 22pt at (50, 20)
  - Tagline "Deploying Expertise That Hasn't Passed Its Sell-By Date." in amber (#c89a30), regular, 11pt at (50, 48)
- Report title section starting at y=110:
  - "Impact Report" in navy, bold, 18pt
  - Institution name in midGrey (#787880), regular, 12pt
  - Date range line: "All time" if both null, else format "1 January 2026 - 31 March 2026" in midGrey, regular, 10pt
- Stats table using `doc.table()` (native pdfkit v0.17+ API):
  - Two columns: Metric | Value
  - Header row with nearWhite (#f7f7fa) background, bold font
  - Data rows: "Contributors" / count, "Active Challenges" / count, "Total Hours Logged" / count
  - Default style: padding 10, bottom border in border colour (#d9d9e0)
- Footer at page bottom (page.height - 40):
  - "Generated {date}" in midGrey, regular, 9pt, right-aligned
  - Date formatted as "24 March 2026" using en-GB locale
- Return the doc (do NOT call doc.end() — the route handler does that)

Brand hex constants (from OKLCH approximations in research):
```
navy: "#1a1d2e", amber: "#c89a30", nearWhite: "#f7f7fa", midGrey: "#787880", border: "#d9d9e0"
```

**B. Add GET route in `packages/server/src/routes/admin.ts`:**

Add a new route BEFORE the `export { router as adminRouter }` line:

```
router.get("/institutions/:slug/report.pdf", async (req, res) => { ... })
```

Route logic (order is critical — all validation BEFORE doc.pipe):

1. Extract `slug` from params. Validate with a slug regex `/^[a-z0-9-]{2,100}$/` (same pattern as institutions.ts). Return 400 if invalid.

2. Extract optional `from` and `to` query params. Parse as Date if present; leave null if absent.

3. Look up institution by slug: `db.select().from(institutions).where(eq(institutions.slug, slug)).limit(1)`. Return 404 if not found.
   NOTE: The adminRouter does NOT filter by isActive — CM should be able to generate reports for inactive institutions too.

4. Fetch contributor assignments for this institution (same pattern as institutions.ts): query `contributorInstitutions` where `institutionId = institution.id`.

5. If `memberIds.length === 0`, return 422 JSON `{ error: "No contributors assigned to this institution" }` — BEFORE any streaming.

6. Query stats with optional date filtering:
   - Challenge interests: `SELECT challengeId FROM challengeInterests WHERE contributorId IN (memberIds)`. If `startDate` is set, add `AND createdAt >= startDate`. If `endDate`, add `AND createdAt <= endDate`. Count unique challengeIds.
   - Hours: `SELECT coalesce(sum(hoursLogged), 0) FROM contributorHours WHERE contributorId IN (memberIds)`. Same date filters on createdAt.

7. Set response headers:
   - `Content-Type: application/pdf`
   - `Content-Disposition: attachment; filename="impact-report-{slug}.pdf"`
   - `Cache-Control: no-store`

8. Import and call `buildInstitutionReport({ institutionName, institutionCity, stats, generatedAt: new Date(), dateRange: { startDate, endDate } })`.

9. `doc.pipe(res)` then `doc.end()`.

Add the import at the top of admin.ts:
```typescript
import { buildInstitutionReport } from "../pdf/institution-report.js";
```

Also add the necessary drizzle imports if not already present (gte, lte from drizzle-orm for date filtering).
