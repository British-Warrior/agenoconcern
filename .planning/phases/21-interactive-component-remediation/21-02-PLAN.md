  <action>
Modify NotificationBell.tsx with the following changes:

**1. aria-expanded on the bell button:**
Add `aria-expanded={open}` to the bell `<button>` element (the one with `aria-label="Notifications"`). This ensures screen readers announce "Notifications, expanded" or "collapsed" on state change.

Also add `aria-haspopup="true"` to the bell button to signal that it controls a popup.

**2. Escape key closes dropdown and returns focus to bell:**
Add a `useRef` for the bell button (`bellRef`). Attach it to the bell `<button>`.

Add a `useEffect` that listens for `keydown` when `open` is true:
```
if (e.key === "Escape") {
  setOpen(false);
  bellRef.current?.focus();
}
```
Clean up the listener when `open` becomes false or on unmount.

**3. Focus management when dropdown opens:**
Add a `useRef` for the dropdown panel (`panelRef`). When `open` becomes true, focus the first focusable element inside the panel. Use a `useEffect` that runs when `open` changes:
```
if (open && panelRef.current) {
  const first = panelRef.current.querySelector<HTMLElement>("button, a[href], [tabindex]:not([tabindex='-1'])");
  first?.focus();
}
```
Use `setTimeout(..., 0)` to wait one tick for the DOM to render the dropdown before querying.

**4. Dropdown panel role and labelling:**
Add `role="menu"` and `aria-label="Notifications"` to the dropdown panel `<div>`. Add `role="menuitem"` to each notification item `<button>` inside the list. Add `role="menuitem"` to the "Mark all read" button as well.

**5. Tab containment within dropdown (lightweight):**
Do NOT use useFocusTrap (that is for modals). Instead, the dropdown is a menu — Tab should close it and move to next page element (standard menu pattern). However, the current implementation already allows Tab to naturally flow through the items since they are `<button>` elements inside the dropdown. The Escape key (added above) provides the close mechanism.

Actually, re-evaluating: the standard pattern for a disclosure dropdown (not a full menu) is that Tab moves through items within the dropdown, and after the last item Tab moves to the next element on the page. This is already the default browser behavior since the notification items are buttons. No special Tab handling needed.

**6. Clean up the outside-click handler:**
The existing `mousedown` listener for closing on outside click should remain. Additionally, the Escape handler (step 2) provides keyboard-equivalent close.

**Summary of attributes to add:**
- Bell button: `aria-expanded={open}`, `aria-haspopup="true"`, `ref={bellRef}`
- Dropdown panel div: `ref={panelRef}`, `role="menu"`, `aria-label="Notifications"`
- "Mark all read" button: `role="menuitem"`
- Each notification item button: `role="menuitem"`

**Summary of effects to add:**
- Escape keydown listener (when open) -> close + focus bell
- Focus first item on open (useEffect with setTimeout 0)
