**In `packages/server/src/routes/admin.ts`**, add these endpoints:

1. `GET /institutions/:id/contributors` — List contributors for an institution. Join `contributorInstitutions` with `contributors` table. Return: id, name (firstName + lastName), role (as "platform status"), lastActivity (see below). Order by name. Validate UUID param.

2. `PUT /contributors/:contributorId/institutions` — **Set assignments** (per user decision: assignment from contributor profile). Validate body with `setContributorInstitutionsSchema` from shared package. Wrap in `db.transaction`: DELETE all existing rows for contributorId, then INSERT new rows for each institutionId (with assignedBy = req.contributor.id). Guard against empty institutionIds array (just delete all = unassign from everything, which is valid). Return 200 with updated list.

3. `GET /contributors` — List all contributors with their institution assignments. Return: id, firstName, lastName, role, institutions (array of {id, name}). This powers the "Unassigned" group in the CM view — contributors with empty institutions array are unassigned.

**Last activity resolution** (Claude's discretion): For each contributor in the institution contributor list, compute last activity as the more recent of:
- MAX(wellbeing_checkins.completed_at) for that contributor
- MAX(circle_notes.created_at) where author_id = contributor
Use two batch queries (not N+1): one for all memberIds against wellbeingCheckins, one against circleNotes, both using `inArray`. Merge in TypeScript taking the later date. Return as ISO string or null ("No activity yet").

**CRITICAL: Guard against empty `inArray`** — if memberIds is empty, skip aggregate queries and return empty array. `inArray(col, [])` generates invalid SQL.

**Batch live stats for institution list**: Update `GET /institutions` to include live stats per institution. Query all `contributorInstitutions` rows in one pass, group by institutionId in TypeScript. For each group, count contributors. Then batch-query challengeInterests and contributorHours for all assigned contributor IDs (deduplicated across institutions). Compute per-institution: contributorCount, challengeCount, hoursTotal. Return these alongside institution fields in the response.

**In `packages/server/src/routes/institutions.ts`**, update the public `GET /api/institutions/:slug` endpoint:
- After fetching the institution record, compute live stats using the same aggregation pattern (get contributor IDs from junction table, aggregate challenges and hours)
- Return `stats: liveStats.contributors > 0 ? liveStats : null` (per user decision: hide stats section when no contributors assigned — null signals this to the frontend)
- Keep statsJson in the DB (per user decision: cache/fallback) but do NOT return it to the client — live stats replace it
- CRITICAL: Guard `inArray` against empty array
