  <action>
Create `packages/server/src/routes/admin.ts` — a new Express router with `authMiddleware` + `requireRole("community_manager")` on every route. Follow the pattern from `challenges.ts` for middleware chaining and error handling.

**Slug generation helper** (at top of file): `function slugify(name: string): string` — lowercase, replace non-alphanumeric with hyphens, collapse multiple hyphens, trim hyphens, truncate to 100 chars. Used only on create.

Endpoints:
1. `GET /institutions` — List all institutions (CM sees active AND inactive). Return array of objects: id, name, slug, description, city, isActive, createdAt. Order by name. No pagination needed for pilot scale (< 20 institutions).

2. `POST /institutions` — Create institution. Validate body with `createInstitutionSchema` from shared package. Auto-generate slug from name using slugify helper. Insert into institutions table. Return 201 with created record. Handle unique slug conflict with 409 error.

3. `PUT /institutions/:id` — Update institution. Validate UUID param (copy UUID_PATTERN from challenger.ts). Validate body with `updateInstitutionSchema`. Update only provided fields. Do NOT allow slug changes (ignore slug in body). Return updated record. 404 if not found.

4. `PATCH /institutions/:id/active` — Toggle active status. Validate UUID param. Validate body with `toggleActiveSchema`. Set isActive to provided value. Return updated record. 404 if not found.

Mount router in `packages/server/src/index.ts`: import `adminRouter` and `app.use("/api/admin", adminRouter)`. Place AFTER auth middleware setup, alongside other route mounts.

IMPORTANT per user decision: No separate page for create/edit — these endpoints support the inline card editing UI. Slug is immutable after creation.
  </action>

`cd packages/server && npx tsc --noEmit` passes.

- `curl -X GET http://localhost:3000/api/admin/institutions` (with valid CM auth cookie) returns 200

- Unauthenticated request returns 401
  </verify>

Four admin API endpoints operational: GET list, POST create, PUT update, PATCH toggle active. All protected by CM role middleware. Slug auto-generated on create, immutable on edit. Mounted at /api/admin.
