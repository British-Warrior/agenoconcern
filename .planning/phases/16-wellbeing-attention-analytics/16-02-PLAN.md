  <action>
    Add a `GET /attention/trend` route in `admin.ts`. This route MUST be registered BEFORE any `GET /attention/:flagId` parameterised routes to prevent "trend" being matched as a flagId. Follow the same placement pattern as `GET /attention/history` (which is already registered before `/:flagId` routes).

    **Route implementation:**

    1. Look up the CM's institution using the same LIMIT 1 pattern as existing attention routes (contributorInstitutions lookup by req.user.id).

    2. Query 12 weeks of flag counts using PostgreSQL generate_series:
    ```typescript
    const trendRows = await db.execute(sql`
      WITH weeks AS (
        SELECT generate_series(
          date_trunc('week', now()) - interval '11 weeks',
          date_trunc('week', now()),
          '1 week'
        ) AS week_start
      )
      SELECT
        to_char(w.week_start, 'YYYY-"W"IW') AS iso_week,
        coalesce(count(f.id)::int, 0) AS count
      FROM weeks w
      LEFT JOIN ithink_attention_flags f
        ON date_trunc('week', f.created_at AT TIME ZONE 'UTC') = w.week_start
        AND f.institution_id = ${institutionId}
      GROUP BY w.week_start
      ORDER BY w.week_start
    `);
    ```

    3. Compute trend direction from the 12-week data:
    - Split 12 weeks into last 4 (current) and previous 4 (comparison). Ignore the oldest 4 weeks (display-only context).
    - Sum flag counts in current period and comparison period.
    - If current > comparison: direction = "Increasing"
    - If current < comparison: direction = "Decreasing"
    - Otherwise: direction = "Stable"

    4. Return JSON:
    ```json
    {
      "weeks": [{ "isoWeek": "2026-W02", "count": 3 }, ...],
      "direction": "Increasing",
      "activeCount": <number of currently unresolved flags>
    }
    ```

    For `activeCount`, query the count of flags where `clearedAt IS NULL` and `institutionId` matches (reuse existing attention query pattern).

    Response type: 200 JSON. Use the existing `sql` import from drizzle-orm (already imported in admin.ts).
