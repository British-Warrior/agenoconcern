    - A `RASCH_METRIC` const array of 29 numbers (raw scores 7-35 mapped to metric scores). Use the official SWEMWBS conversion table values. The table is published by University of Warwick. The 29 values (raw 7 through 35) are:
      7.00, 9.51, 11.25, 12.40, 13.33, 14.08, 14.75, 15.32, 15.84, 16.36, 16.88, 17.43, 17.98, 18.59, 19.25, 19.98, 20.73, 21.54, 22.35, 23.21, 24.11, 25.03, 26.02, 27.03, 28.13, 29.31, 30.70, 32.55, 35.00
    - `export function rawToMetric(rawScore: number): number` — validates 7-35 range, returns `RASCH_METRIC[rawScore - 7]`
    - `export type WellbeingBandLabel = "Low" | "Typical" | "High"`
    - `export function metricToBand(metricScore: number): WellbeingBandLabel` — Uses metric-score band boundaries derived from the Rasch table. Raw score 19 maps to metric 18.59 and raw 20 maps to 19.25, so the Low/Typical boundary is 19.25 (first metric score in the Typical range). Raw score 27 maps to 28.13 and raw 28 maps to 29.31, so the Typical/High boundary is 29.31. Implementation:
      - `metricScore < 19.25` returns "Low"
      - `metricScore < 29.31` returns "Typical"
      - otherwise returns "High"

    Add a comment at the top: `// Source: University of Warwick SWEMWBS Raw Score to Metric Score Conversion Table`

    - If `data.wellbeingBand` is a string ("Low", "Typical", or "High"):
      - Draw a row with same styling as stat rows (alternating bg based on index, which will be index 3 = odd = LIGHT_GREY background)
      - Left label: "Wellbeing Band" in navy regular 11pt
      - Right value: the band label (e.g. "Typical") in navy bold 11pt (use navy `#1a1d2e` for all bands per research recommendation — fully neutral)
    - If `data.wellbeingBand` is `null` and `data.wellbeingMessage` is set:

    - If `data.wellbeingBand` is undefined (not computed, e.g. no wellbeing data at all): skip the row entirely.

    **3. Wellbeing band query in admin.ts** — In the existing PDF route handler (`GET /institutions/:slug/report.pdf` in `admin.ts`), BEFORE calling `buildInstitutionReport`, add wellbeing band computation:

    Import `rawToMetric`, `metricToBand`, and `WellbeingBandLabel` from `../lib/swemwbs-rasch.js`.

    Query the most recent consented check-in per contributor at this institution using raw SQL (Drizzle cannot express DISTINCT ON):
    ```typescript
    const wellbeingRows = await db.execute(sql`
      SELECT DISTINCT ON (wc.contributor_id)
        wc.wemwbs_score
      FROM wellbeing_checkins wc
      INNER JOIN contributor_institutions ci
        ON ci.contributor_id = wc.contributor_id
      WHERE ci.institution_id = ${institutionId}
        AND wc.institutional_reporting = true
      ORDER BY wc.contributor_id, wc.completed_at DESC
    `);
    ```

    Apply k-anonymity (k=5):
    - If `wellbeingRows.rows.length < 5`: set `wellbeingBand = null` and `wellbeingMessage = "Wellbeing data not available — fewer than 5 contributors have shared their wellbeing data for this report period."`
    - If >= 5: compute modal band:
      - For each row, call `rawToMetric(row.wemwbs_score)` then `metricToBand(metricScore)`
      - Count occurrences of each band
      - Modal band = most frequent. Tie-break order: Typical > High > Low (prefer Typical)
      - Set `wellbeingBand` to the modal band label

    Pass `wellbeingBand` and `wellbeingMessage` (if suppressed) to `buildInstitutionReport` in the `ReportData` object.

    IMPORTANT: The `institutionId` variable should already be available in the PDF route handler from the existing institution lookup. Verify this — the existing handler looks up the institution by slug and has the institution object. Use `institution.id`.

    IMPORTANT: Do NOT leak the exact count of consented contributors when below k — the suppression message uses the generic "fewer than 5" wording, not the actual count.
