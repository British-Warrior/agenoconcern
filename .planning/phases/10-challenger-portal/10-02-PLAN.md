**API client (packages/web/src/api/challenger.ts):**
Follow the exact pattern from `packages/web/src/api/challenges.ts`. Import `apiClient` from `./client.js`. Export an object `challengerApi` with:
- `register(data: RegisterChallengerInput)` → POST `/api/challenger/register` (returns contributor + org)
- `getOrg()` → GET `/api/challenger/me` (returns ChallengerOrg)
- `getMyChallenges()` → GET `/api/challenger/challenges` (returns ChallengerChallenge[])
- `getChallengeDetail(id: string)` → GET `/api/challenger/challenges/${id}` (returns ChallengerChallengeDetail)
- `submitChallenge(data: SubmitChallengerChallengeInput)` → POST `/api/challenger/challenges` (returns ChallengerChallenge)

Import types from `@iu/shared/types/challenger.js` and input types from `@iu/shared/schemas/challenger.js`.

**React Query hooks (packages/web/src/hooks/useChallenger.ts):**
Follow the exact pattern from `useChallenges.ts`:
- `useRegisterChallenger()` → useMutation calling `challengerApi.register`, onSuccess invalidates `["auth", "me"]` query
- `useChallengerOrg()` → useQuery with queryKey `["challenger", "org"]`
- `useMyChallenges()` → useQuery with queryKey `["challenger", "challenges"]`
- `useChallengeDetail(id: string)` → useQuery with queryKey `["challenger", "challenges", id]`, enabled when id is truthy
- `useSubmitChallenge()` → useMutation calling `challengerApi.submitChallenge`, onSuccess invalidates `["challenger", "challenges"]` query

**ChallengerRoute guard (packages/web/src/components/layout/ChallengerRoute.tsx):**
Follow ProtectedRoute pattern. Uses `useAuth()` hook:
- If loading: show the same spinner as ProtectedRoute
- If not authenticated: redirect to `/challenger/register` (not `/login` — challengers have their own registration)
- If `contributor.role !== "challenger" && contributor.role !== "admin"`: redirect to `/dashboard`
- Otherwise: render `<Outlet />`

**ProtectedRoute update (packages/web/src/components/layout/ProtectedRoute.tsx):**
Add a redirect for challenger role. After the onboarding redirect block, before `return <Outlet />`:
```typescript
// Redirect challenger-role users to their portal
if (contributor?.role === "challenger" && !location.pathname.startsWith("/challenger")) {
  return <Navigate to="/challenger" replace />;
}
```
This prevents challengers from accidentally landing on the contributor dashboard.

**ROUTES constants (packages/web/src/lib/constants.ts):**
Add challenger routes to the ROUTES object:
```typescript
CHALLENGER: "/challenger",
CHALLENGER_REGISTER: "/challenger/register",
CHALLENGER_SUBMIT: "/challenger/submit",
```

**App.tsx routing (packages/web/src/App.tsx):**
Add imports for ChallengerRoute, ChallengerRegister, ChallengerDashboard, SubmitChallenge, ChallengeDetail.

Add a public route for registration (outside ProtectedRoute):
```tsx
<Route path="/challenger/register" element={<ChallengerRegister />} />
```

Add a ChallengerRoute-guarded group (outside ProtectedRoute, since ChallengerRoute handles its own auth):
```tsx

  <Route path="/challenger" element={<ChallengerDashboard />} />
  <Route path="/challenger/submit" element={<SubmitChallenge />} />
  <Route path="/challenger/challenges/:id" element={<ChallengeDetail />} />
</Route>
```
