**Shared Zod schemas (packages/shared/src/schemas/challenger.ts):**
```typescript
import { z } from "zod";

export const registerChallengerSchema = z.object({
  contactName: z.string().min(2, "Name must be at least 2 characters"),
  email: z.string().email("Valid email required"),
  password: z.string().min(8, "Password must be at least 8 characters"),
  organisationName: z.string().min(2, "Organisation name required"),
  organisationType: z.string().min(1, "Organisation type required").max(100),
});

export const submitChallengerChallengeSchema = z.object({
  title: z.string().min(5, "Title must be at least 5 characters").max(200),
  brief: z.string().min(20, "Brief must be at least 20 characters"),
  domain: z.string().min(1, "Domain required"),
  skillsNeeded: z.array(z.string()).min(1, "At least one skill required"),
  type: z.enum(["community", "premium", "knowledge_transition"]),
  deadline: z.string().datetime().optional(),
  maxMembers: z.number().int().min(3).max(7).optional(),
});

export type RegisterChallengerInput = z.infer<typeof registerChallengerSchema>;
export type SubmitChallengerChallengeInput = z.infer<typeof submitChallengerChallengeSchema>;
```

**Challenger routes (packages/server/src/routes/challenger.ts):**

Create a new Express Router with these endpoints:

**1. POST /register** (public, no auth)
- Validate body with `registerChallengerSchema.safeParse(req.body)` — return 400 on failure
- Check for existing contributor with same email — return 409 if exists
- In a `db.transaction`:
  - Insert into `contributors`: name=contactName, email=email.toLowerCase(), passwordHash=argon2.hash(password), authProvider="email", role="challenger", status="active" (NOT onboarding — challengers skip CV upload)
  - Insert into `challengerOrganisations`: name=organisationName, contactEmail=email.toLowerCase(), contributorId=contributor.id, organisationType=organisationType
- After transaction: call `setAuthCookies(res, contributor)` from auth.service.ts to set JWT session cookies
- Return 201 with `{ contributor: { id, name, email, role }, organisation: { id, name, organisationType } }`

**2. GET /me** (authMiddleware + requireRole("challenger"))
- Query `challengerOrganisations` WHERE `contributorId = req.contributor.id`
- Return 200 with org data or 404 if not found

**3. POST /challenges** (authMiddleware + requireRole("challenger"))
- Validate body with `submitChallengerChallengeSchema.safeParse(req.body)`
- Insert into `challenges`: ...validated fields, createdBy=req.contributor.id, status="draft" (CRITICAL: not "open")
- Return 201 with the created challenge

**4. GET /challenges** (authMiddleware + requireRole("challenger"))
- Query `challenges` WHERE `createdBy = req.contributor.id`, ordered by `createdAt DESC`
- LEFT JOIN `circles` on `circles.challengeId = challenges.id` to get circle status and member count
- Use `sql<number>\`COUNT(...)\`` or a subquery for member count from `circle_members`
- Return 200 with array of ChallengerChallenge objects (include circle data if circle exists)

**5. GET /challenges/:id** (authMiddleware + requireRole("challenger"))
- Query single challenge WHERE `id = :id AND createdBy = req.contributor.id` — return 404 if not found (ownership check!)
- LEFT JOIN circles, count members, LEFT JOIN circle_resolutions, LEFT JOIN resolution_ratings
- Return 200 with ChallengerChallengeDetail including circle members, resolution, and rating data

**Mount in express-app.ts:**
- Import challengerRoutes from `./routes/challenger.js`
- Add `app.use("/api/challenger", challengerRoutes)` alongside other route mounts (before error handler)
