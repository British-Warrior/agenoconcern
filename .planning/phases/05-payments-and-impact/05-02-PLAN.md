Create `packages/server/src/routes/payments.ts`. Follow the pattern from existing routes (e.g., circles.ts, challenges.ts).

Import Router from express, authMiddleware, getDb, schema tables (paymentTransactions, contributorHours, contributorProfiles, challenges, circleMembers), Stripe service functions, Zod schemas from shared, drizzle operators (eq, and, desc).

Export `paymentRoutes` Router and `webhookHandler` as a separate named export (the handler is registered directly on `app` in express-app.ts, not through the router, because it needs express.raw()).

**Router endpoints (all require authMiddleware, CM role guard unless noted):**

1. `POST /retainer` (CM only):
   - Validate body with createRetainerSchema.
   - Look up contributor's stripeAccountId from contributorProfiles WHERE contributorId matches.
   - If no stripeAccountId, return 400 "Contributor has no Stripe Connect account".
   - Call createRetainerSubscription with { challengerCustomerId: body.challengerCustomerId, retainerAmountPence: body.amountPence, contributorAccountId: profile.stripeAccountId, challengeId: body.challengeId, contributorId: body.contributorId }.
   - Insert paymentTransaction: status='held', amountPence=Math.floor(body.amountPence * 0.75), totalAmountPence=body.amountPence, stripeSubscriptionId=subscription.id, paymentType='retainer'.
   - Return 201 with { subscriptionId: subscription.id }.

2. `POST /stipend` (CM only):
   - Validate body with chargeStipendSchema.
   - Call chargeStipend with { totalAmountPence: body.amountPence, challengerCustomerId: body.challengerCustomerId, paymentMethodId: body.paymentMethodId, circleId: body.circleId, challengeId: body.challengeId }.
   - Insert paymentTransaction: status='held', amountPence=Math.floor(body.amountPence * 0.75), totalAmountPence=body.amountPence, stripePaymentIntentId=result.paymentIntentId, stripeChargeId=result.chargeId, paymentType='stipend'. NOTE: stripeChargeId MUST be stored here — it is required for the release transfer.
   - Return 201 with { paymentIntentId: result.paymentIntentId }.

3. `POST /stipend/release` (CM only):
   - Validate body with releaseStipendSchema.
   - Find paymentTransaction WHERE circleId matches AND paymentType='stipend' AND status='held'. If not found, return 404 "No held stipend for this circle".
   - Look up contributor's stripeAccountId from contributorProfiles.
   - Call releaseStipendTransfer with { amountPence: txn.amountPence, contributorAccountId: profile.stripeAccountId, circleId: body.circleId, sourceChargeId: txn.stripeChargeId }. The sourceChargeId is the ch_... value stored at stipend creation.
   - Update paymentTransaction: set status='transferred', stripeTransferId=transfer.id, transferredAt=new Date(), updatedAt=new Date().
   - Return 200 with { transferId: transfer.id }.

4. `POST /sme-subscription` (CM only):
   - Validate body with createSmeSubscriptionSchema.
   - Same pattern as retainer but call createSmeSubscription with the interval parameter and paymentType='sme_subscription'.
   - Return 201 with { subscriptionId: subscription.id }.

5. `POST /hours` (authenticated contributor, no CM guard):
   - Validate body with logHoursSchema.
   - Verify caller is a member of the specified circle: query circleMembers WHERE circleId AND contributorId = req.user.id. If not member, return 403.
   - Insert into contributorHours: contributorId=req.user.id, circleId, hoursLogged, description, isPaid (default false).
   - Return 201 with the inserted row.

6. `GET /my-transactions` (authenticated contributor, no CM guard):
   - Query paymentTransactions WHERE contributorId = req.user.id, ORDER BY createdAt DESC.
   - Return 200 with the array.

**Webhook handler (exported separately, NOT on the router):**

Export `async function webhookHandler(req: Request, res: Response)`:
- Extract signature: `req.headers['stripe-signature'] as string`.
- Try constructWebhookEvent(req.body, sig). On error, return 400 with error message.
- Idempotency guard: query paymentTransactions WHERE stripeEventId = event.id. If found, return `res.json({ received: true })` immediately (200).
- Switch on event.type:
  - `'invoice.paid'`: Extract invoice from event.data.object. Get metadata (challengeId, contributorId, paymentType) from invoice.subscription_details?.metadata or invoice.metadata. Insert paymentTransaction with status='transferred' (destination charges auto-transfer), stripeSubscriptionId from invoice.subscription, stripeEventId=event.id, amountPence=Math.floor(invoice.amount_paid * 0.75), totalAmountPence=invoice.amount_paid. Set transferredAt=new Date().
  - `'invoice.payment_failed'`: Log to console. Find matching paymentTransaction by stripeSubscriptionId if exists, update status='failed'.
  - `'customer.subscription.deleted'`: Log to console. No further action for MVP.
- Always return `res.json({ received: true })` with 200.

Wrap handler body in try/catch — on unexpected errors, log and return 500.

   `GET /summary` (authenticated contributor):
   - Run 3 parallel queries via Promise.all:
     a. Challenges participated: SELECT from challengeInterests INNER JOIN challenges WHERE contributorId = req.user.id. LEFT JOIN circleMembers to get circleId if the contributor is in a circle for that challenge. Select challenge id, title, domain, type, status, and circleId.

     c. Earnings: SELECT from paymentTransactions WHERE contributorId = req.user.id AND status = 'transferred'. Compute totalEarningsPence (sum amountPence) in JS. Map rows to ImpactEarning shape.
   - Return ImpactSummary shape with wellbeingTrajectory: [].

   `GET /challenger` (authenticated):
   - A challenger is the creator of challenges. Query challenges WHERE createdBy = req.user.id.
   - For each challenge, LEFT JOIN circleResolutions (via the circle linked to the challenge — join circles WHERE challengeId, then circleResolutions WHERE circleId). LEFT JOIN resolutionRatings on resolutionId.
   - Return ChallengerImpact shape: array of ChallengerChallenge objects with resolution and rating nested if they exist.
   - If no challenges found, return empty array (not an error).
