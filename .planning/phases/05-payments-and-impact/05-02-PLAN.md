  <action>
1. Create `packages/shared/src/types/impact.ts`:
   - `ImpactSummary` interface: { challengesParticipated: Array<{ id, title, domain, type, status, circleId? }>, totalHours: number, paidHours: number, unpaidHours: number, totalEarningsPence: number, earnings: Array<{ id, paymentType, amountPence, status, createdAt }>, wellbeingTrajectory: never[] (empty — Phase 6) }
   - `ChallengerImpact` interface: { challenges: Array<{ id, title, status, resolution?: { problemSummary, recommendations, submittedAt }, rating?: { rating, feedback, createdAt } }> }
   - Re-export from `packages/shared/src/index.ts`

2. Create `packages/server/src/routes/impact.ts`:
   - Export `impactRoutes` Router. All routes require authMiddleware.

   - `GET /summary` (authenticated contributor):
     Run 3 parallel queries via Promise.all:
       a. Challenges participated: JOIN challengeInterests to challenges WHERE contributorId = caller. Include circle membership info by LEFT JOIN circleMembers. Select id, title, domain, type, status, and circleId if member exists.
       b. Hours: SELECT from contributorHours WHERE contributorId = caller. Sum total, paid (isPaid=true), unpaid (isPaid=false).
       c. Earnings: SELECT from paymentTransactions WHERE contributorId = caller AND status = 'transferred'. Sum amountPence.
     Return ImpactSummary shape with wellbeingTrajectory: [].

   - `GET /challenger` (authenticated, role = challenger logic):
     A challenger is identified by challenges.createdBy = caller's ID.
     Query challenges WHERE createdBy = caller. For each, LEFT JOIN circleResolutions and resolutionRatings.
     Return ChallengerImpact shape.

   - `POST /hours` (authenticated contributor):
     Validate body with logHoursSchema (from shared). Verify caller is a member of the specified circle (check circleMembers). Insert into contributorHours with contributorId = caller. Return 201.
     NOTE: This endpoint may also exist in payments.ts from Plan 01. If Plan 01 already created it there, move it here to impact.ts instead (impact is the better semantic home). If both exist, keep only the one in impact.ts.

3. Register in `packages/server/src/express-app.ts`:
   - Import impactRoutes from './routes/impact.js'
   - Add `app.use('/api/impact', impactRoutes)` after payment routes
  </action>
