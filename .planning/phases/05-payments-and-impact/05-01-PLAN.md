1. Extend `packages/server/src/services/stripe.service.ts` with new exported functions:

   - `createRetainerSubscription(params: { challengerCustomerId, retainerAmountPence, contributorAccountId, challengeId, contributorId })`: Creates a Stripe Price (monthly, GBP) then creates Subscription with `application_fee_percent: 25`, `transfer_data: { destination: contributorAccountId }`, metadata: { challengeId, contributorId, paymentType: 'retainer' }. Returns the Subscription object.

   - `createSmeSubscription(params: { challengerCustomerId, amountPence, contributorAccountId, challengeId, contributorId, interval: 'month'|'year' })`: Same pattern as retainer but with configurable interval and paymentType: 'sme_subscription' in metadata.

   - `chargeStipend(params: { totalAmountCents, challengerCustomerId, paymentMethodId, circleId, challengeId })`: Creates PaymentIntent with `confirm: true`, `transfer_group: circleId`, currency 'gbp', metadata: { circleId, challengeId, paymentType: 'stipend' }. Normal capture (NOT capture_method=manual). Returns PaymentIntent.

   - `releaseStipendTransfer(params: { amountPence, contributorAccountId, circleId, sourceTransaction })`: Creates Transfer with `destination: contributorAccountId`, `transfer_group: circleId`, `source_transaction: sourceTransaction`, currency 'gbp'. Returns Transfer.

   - `constructWebhookEvent(rawBody: Buffer, signature: string)`: Calls `stripe.webhooks.constructEvent(rawBody, signature, env.STRIPE_WEBHOOK_SECRET)`. Returns Stripe.Event.

   Keep existing functions (createConnectAccount, createAccountLink, getAccountStatus) unchanged.

2. Create `packages/server/src/routes/payments.ts`:
   - Import Router from express, authMiddleware, requireRole('community_manager') guard, getDb, schema tables, stripe service functions, zod schemas from shared.
   - Export `paymentRoutes` Router.

   Endpoints (all CM-only except webhook):

   - `POST /retainer` (CM): Validate body with createRetainerSchema. Look up contributor's stripeAccountId from contributorProfiles. Call createRetainerSubscription. Insert paymentTransaction row with status='held', stripeSubscriptionId set. Return 201 with subscription ID.

   - `POST /stipend` (CM): Validate body with chargeStipendSchema. Look up challenge and circle. Call chargeStipend. Insert paymentTransaction row with status='held', stripePaymentIntentId set, amountPence = Math.floor(totalAmount * 0.75). Return 201 with paymentIntent ID.

   - `POST /stipend/release` (CM): Validate body with releaseStipendSchema. Find 'held' paymentTransaction for circleId with paymentType='stipend'. Look up contributor's stripeAccountId. Call releaseStipendTransfer with source_transaction = the payment intent's latest_charge (stored as stripePaymentIntentId — note: for separate charges, store the charge ID, not PI ID, OR use `source_transaction` which accepts PI ID). Update paymentTransaction to status='transferred', set stripeTransferId and transferredAt. Return 200.

   - `POST /sme-subscription` (CM): Validate body with createSmeSubscriptionSchema. Same pattern as retainer with interval parameter. Return 201.

   - `POST /hours` (authenticated contributor): Validate body with logHoursSchema. Verify caller is a circle member. Insert contributorHours row. Return 201.

   - `GET /my-transactions` (authenticated contributor): Return paymentTransactions for current user, ordered by createdAt desc.

   - Webhook handler (NO auth middleware — Stripe sends directly):
     Create a separate handler function `webhookHandler` that receives raw body.
     Verify signature via constructWebhookEvent.
     Idempotency guard: check for existing paymentTransaction with matching stripeEventId. If found, return 200 immediately.
     Handle events:
       - `invoice.paid`: Extract metadata (challengeId, contributorId, paymentType) from invoice. Insert or update paymentTransaction with status='transferred' (destination charges auto-transfer). Set stripeEventId.
       - `invoice.payment_failed`: Find matching paymentTransaction by stripeSubscriptionId, update status to 'failed'.
       - `customer.subscription.deleted`: Log — no action needed for MVP.
     Always return `res.json({ received: true })` with 200 status.
