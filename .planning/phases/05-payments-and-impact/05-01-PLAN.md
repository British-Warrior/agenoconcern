  <action>
Extend `packages/server/src/services/stripe.service.ts` with 5 new exported functions. Keep all existing functions (createConnectAccount, createAccountLink, getAccountStatus) unchanged.

1. `createRetainerSubscription(params: { challengerCustomerId: string, retainerAmountPence: number, contributorAccountId: string, challengeId: string, contributorId: string })`:
   - Create a Stripe Price: `stripe.prices.create({ unit_amount: retainerAmountPence, currency: 'gbp', recurring: { interval: 'month' }, product_data: { name: 'Knowledge Transition Retainer - Challenge ' + challengeId } })`
   - Create a Stripe Subscription: `stripe.subscriptions.create({ customer: challengerCustomerId, items: [{ price: price.id }], application_fee_percent: 25, transfer_data: { destination: contributorAccountId }, metadata: { challengeId, contributorId, paymentType: 'retainer' } })`
   - Return the Subscription object.

2. `createSmeSubscription(params: { challengerCustomerId: string, amountPence: number, contributorAccountId: string, challengeId: string, contributorId: string, interval: 'month' | 'year' })`:
   - Same pattern as retainer but with `params.interval` for the recurring interval and `paymentType: 'sme_subscription'` in metadata. Product name: `'SME Subscription - Challenge ' + challengeId`.
   - Return the Subscription object.

3. `chargeStipend(params: { totalAmountPence: number, challengerCustomerId: string, paymentMethodId: string, circleId: string, challengeId: string })`:
   - Create PaymentIntent: `stripe.paymentIntents.create({ amount: totalAmountPence, currency: 'gbp', customer: challengerCustomerId, payment_method: paymentMethodId, confirm: true, automatic_payment_methods: { enabled: true, allow_redirects: 'never' }, transfer_group: circleId, metadata: { circleId, challengeId, paymentType: 'stipend' } })`
   - Do NOT use `capture_method: 'manual'` — card auth expires in 7 days. Normal immediate capture is required.
   - Extract charge ID: `const chargeId = paymentIntent.latest_charge as string` — this is the `ch_...` ID needed for `source_transaction`.
   - Return `{ paymentIntentId: paymentIntent.id, chargeId }`.

4. `releaseStipendTransfer(params: { amountPence: number, contributorAccountId: string, circleId: string, sourceChargeId: string })`:
   - Create Transfer: `stripe.transfers.create({ amount: amountPence, currency: 'gbp', destination: contributorAccountId, transfer_group: circleId, source_transaction: sourceChargeId, metadata: { circleId } })`
   - `source_transaction` MUST be a Charge ID (`ch_...`), NOT a PaymentIntent ID (`pi_...`). The `sourceChargeId` param is the value stored in `paymentTransactions.stripeChargeId`.
   - Return the Transfer object.

5. `constructWebhookEvent(rawBody: Buffer, signature: string)`:
   - Import getEnv from config/env.
   - Call `stripe.webhooks.constructEvent(rawBody, signature, getEnv().STRIPE_WEBHOOK_SECRET)`.
   - Return the Stripe.Event object.
   - Let errors propagate (caller handles).
