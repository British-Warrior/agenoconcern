   Layout: Responsive grid — single column on mobile, 2 columns on lg breakpoint. Page title "My Impact" with brief intro text.

   Five sections as Card components:

   a. **Challenges Participated** (IMPT-01):
      - Header: "Challenges" with count badge.
      - List of challenge titles with domain badge(s) and status indicator (colored dot or badge: open=green, closed=grey, archived=amber).
      - Empty state: "You haven't participated in any challenges yet."

   b. **Hours Contributed** (IMPT-02):
      - Large number display for totalHours.
      - Below: "{paidHours} paid | {unpaidHours} volunteered" breakdown.
      - "Log Hours" button that toggles an inline form:
        - Circle selector: dropdown populated from challengesParticipated that have circleId (filter for circles the user is in). If no circles, show disabled "Join a circle first".
        - Hours input: number, min 1, max 24.
        - Description: optional textarea, max 500 chars.
        - isPaid: checkbox labeled "Paid work".
        - Submit button calling useLogHours mutation. On success, close form and show success toast (if sonner/toast exists) or just close.
      - Empty state for no hours: "No hours logged yet."

   c. **Earnings** (IMPT-03):
      - Large formatted number: format totalEarningsPence / 100 using `new Intl.NumberFormat('en-GB', { style: 'currency', currency: 'GBP' }).format(...)`.
      - Below: list of recent earnings (last 5-10) showing paymentType badge (retainer=blue, stipend=green, sme_subscription=purple) and date formatted with toLocaleDateString.
      - Empty state: "No earnings yet. Earnings will appear here when you complete paid challenges."

   d. **Unpaid Contribution Recognised** (IMPT-04):
      - Display unpaidHours prominently.
      - Affirming text: "{unpaidHours} hours of mentoring, volunteering, and community contribution formally recognised."
      - If 0 hours: "Your volunteer contributions will be recognised here."

   e. **Wellbeing Trajectory** (IMPT-05 placeholder):
      - Card with muted/disabled visual style (opacity-60 or bg-muted).
      - "Coming Soon" badge or header.
      - Text: "Your wellbeing journey will appear here once check-ins are available."
      - Do NOT add any data fetching or fake data.

   Loading state: Skeleton cards (simple div with animate-pulse, match card dimensions).
   Error state: Error alert using existing Alert component.
