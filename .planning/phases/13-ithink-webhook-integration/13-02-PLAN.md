  <files>packages/server/src/routes/webhooks.ts</files>
  <action>
Create `packages/server/src/routes/webhooks.ts` with the following exports and structure:

**Helper: verifySignature(rawBody: Buffer, incomingSig: string, secret: string): boolean**
- Use `createHmac("sha256", secret).update(rawBody).digest("hex")` from `node:crypto`
- Convert both incoming and expected to Buffers
- Check `incomingBuf.length !== expectedBuf.length` — return false if mismatch (prevents timingSafeEqual throw)
- Return `timingSafeEqual(incomingBuf, expectedBuf)` — copy exact pattern from api-key-auth.ts

**Helper: verifyWithDualSecret(rawBody: Buffer, sig: string): boolean**
- Get `ITHINK_WEBHOOK_SECRET` and `ITHINK_WEBHOOK_SECRET_PREV` from getEnv()
- If primary verifies, return true
- If prev exists and prev verifies, return true
- Return false

**Export: ithinkWebhookHandler(req: Request, res: Response): Promise<void>**

Execute in this EXACT order (non-negotiable per research):

1. Extract `x-ithink-signature` header — if missing or not string, return 401 "Missing signature"
2. Verify HMAC via verifyWithDualSecret(req.body as Buffer, sig) — if false, return 401 "Invalid signature"
3. Parse raw body: `JSON.parse((req.body as Buffer).toString("utf8"))` — if fails, return 400 "Invalid JSON"
4. Timestamp window: extract `timestamp` from parsed object, check `Math.abs(Math.floor(Date.now() / 1000) - parsed.timestamp) > 300` — if true or timestamp missing/non-numeric, return 401 "Request timestamp outside acceptable window"
5. Zod validate with `ithinkWebhookPayloadSchema.safeParse()` — if fails, return 400 with flattened errors
6. Idempotency: SELECT from webhookDeliveries WHERE deliveryId matches — if found, return 200 `{ received: true }` (no flag written)
7. Resolve contributor by email from contributors table — if not found, return 422 "Unknown contributor or institution"
8. Resolve institution by slug from institutions table — if not found, return 422 "Unknown contributor or institution"
9. Check assignment in contributorInstitutions junction table (contributorId + institutionId) — if not found, return 403 "Contributor not assigned to this institution"
10. INSERT into ithinkAttentionFlags: contributorId, institutionId, deliveryId, signalType, cohortSize (nullable), flaggedCount (nullable)
11. INSERT into webhookDeliveries: deliveryId, source "ithink"
12. Return 200 `{ received: true }`

Import from: node:crypto (createHmac, timingSafeEqual), drizzle-orm (eq, and), db/schema.ts (all needed tables), config/env.ts (getEnv), db/index.ts (getDb), shared package (ithinkWebhookPayloadSchema)
