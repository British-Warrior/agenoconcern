Create `webhook.service.ts`:
- Import `createHmac` from `react-native-quick-crypto` (NOT node:crypto — this is React Native)
- Import UUID generator (use the platform's preferred method — `crypto.randomUUID()` if available, or `uuid` package)
- Read `IU_WEBHOOK_URL` and `ITHINK_WEBHOOK_SECRET` from environment/config (use whatever config pattern iThink already uses — do NOT invent a new one)

Export `dispatchAttentionFlag(params)`:
- params: `{ contributorEmail: string, institutionSlug: string, signalType: string, cohortSize?: number, flaggedCount?: number }`
- Build payload object: `{ deliveryId: generateUUID(), timestamp: Math.floor(Date.now() / 1000), ...params }`
- Stringify payload: `const body = JSON.stringify(payload)`
- Sign: `createHmac("sha256", ITHINK_WEBHOOK_SECRET).update(body).digest("hex")`
- Fire-and-forget fetch (no await in caller):
  ```
  fetch(IU_WEBHOOK_URL, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "X-IThink-Signature": signature,
      "X-IThink-Timestamp": String(payload.timestamp),
      "X-IThink-Delivery": payload.deliveryId,
    },
    body,
  }).catch(err => console.warn("[webhook] dispatch failed:", err));
  ```
- The .catch() is intentional — fire-and-forget, no retry at this phase

Wire into screening handler:
- Find the function that runs after a screening completes with a concern flag
- Add a call to `dispatchAttentionFlag(...)` with the relevant data
- The call should NOT be awaited — it's fire-and-forget
