  <decision>STATE.md flags the payload field names as a prerequisite that must be agreed before planning. The research proposes these exact field names for the Zod schema and DB columns. Confirm or correct before proceeding.</decision>
  <context>
Any field name deviation after this point breaks the Zod schema, DB columns, and both the IU receiver and iThink dispatcher. This is the one-time contract agreement.

Proposed payload contract:
- deliveryId (string, UUID) — unique per dispatch, used for idempotency
- timestamp (number, Unix epoch seconds) — used for 5-minute replay window
- contributorEmail (string, email) — identifies the contributor
- institutionSlug (string) — identifies the sending institution
- signalType (enum: "attention_flag") — extensible later
- cohortSize (number, optional) — screening cohort size
- flaggedCount (number, optional) — number flagged in cohort

Headers:
- X-IThink-Signature — HMAC-SHA256 hex digest
- X-IThink-Timestamp — Unix epoch seconds (redundant with body, but standard practice)
- X-IThink-Delivery — delivery ID (redundant with body, but useful for logging)
  </context>

- Import `postgres` from "postgres", connect via DATABASE_URL
- Create `ithink_signal_type` enum: `DO $$ BEGIN IF NOT EXISTS (SELECT 1 FROM pg_type WHERE typname = 'ithink_signal_type') THEN CREATE TYPE ithink_signal_type AS ENUM ('attention_flag'); END IF; END $$;` (CREATE TYPE IF NOT EXISTS is not valid PostgreSQL — use the DO block pattern)
- Create `webhook_deliveries` table (IF NOT EXISTS): id UUID PK defaultRandom, delivery_id VARCHAR(255) NOT NULL UNIQUE, source VARCHAR(50) NOT NULL DEFAULT 'ithink', received_at TIMESTAMPTZ NOT NULL DEFAULT NOW(), processed_at TIMESTAMPTZ
- Create `ithink_attention_flags` table (IF NOT EXISTS): id UUID PK defaultRandom, contributor_id UUID NOT NULL REFERENCES contributors(id) ON DELETE CASCADE, institution_id UUID NOT NULL REFERENCES institutions(id) ON DELETE CASCADE, delivery_id VARCHAR(255) NOT NULL UNIQUE, signal_type ithink_signal_type NOT NULL, cohort_size INTEGER, flagged_count INTEGER, cleared_by UUID REFERENCES contributors(id) ON DELETE SET NULL, cleared_at TIMESTAMPTZ, follow_up_notes TEXT, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(), updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
- Mark migration: INSERT INTO drizzle.__drizzle_migrations (hash, created_at) VALUES ('0006_ithink-webhook-tables', {Date.now()}) ON CONFLICT DO NOTHING
- Close connection with `await sql.end()`
