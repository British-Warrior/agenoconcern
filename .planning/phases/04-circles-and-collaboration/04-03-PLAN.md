       - Props: circleId, existingResolution (nullable — for edit mode)
       - 5 text fields with labels and placeholders:
         a. Problem Summary (required, textarea, min 10 chars)
         b. Recommendations (required, textarea, min 10 chars)
         c. Evidence (required, textarea, min 10 chars)
         d. Dissenting Views (optional, textarea)
         e. Implementation Notes (optional, textarea)
       - If existingResolution provided, pre-fill fields (edit mode)

       - On submit: call useSubmitResolution (POST) or useUpdateResolution (PUT) depending on mode
       - Client-side validation with Zod submitResolutionSchema — show field-level errors
       - Success: show toast/alert "Resolution submitted" and switch to view mode
       - Only show form when circle status is "active" or "submitted" (not "completed")

    3. Create packages/web/src/components/circles/ResolutionCard.tsx:

       - If resolution exists: render all 5 fields in a structured card layout with section headers
       - If resolution exists AND isChallenger AND no rating yet AND circle status is "submitted":
         Show rating UI inline below resolution:
         - Star rating 1-5 (clickable star icons or numbered buttons — use simple numbered buttons for accessibility: "1" "2" "3" "4" "5" with selected state)
         - Feedback textarea (optional)
         - "Submit Rating" button → calls useRateResolution mutation
       - If rating exists: show rating value (e.g., "Rated: 4/5") and feedback text below resolution
       - If circle status is "submitted" and current user is a member (not challenger): show "Awaiting challenger rating" message

    4. Create packages/web/src/components/circles/AddMemberModal.tsx:
       - Props: isOpen, onClose, circleId
       - Simple form: text input for contributor ID (at pilot scale, CM knows the ID or we could search — for MVP, just accept UUID input)
       - "Add Member" button calls useAddMember mutation
       - On success: close modal, show success toast
       - On error: show error message (e.g., "Contributor has reached their maximum number of Circles")
