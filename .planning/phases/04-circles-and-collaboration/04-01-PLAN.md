       - socialChannelEnum: pgEnum("social_channel", ["whatsapp", "slack", "discord", "teams", "signal"])
       - circles table: id, challengeId (FK challenges, onDelete restrict), createdBy (FK contributors, onDelete restrict), status (circleStatusEnum, default "forming"), socialChannel (socialChannelEnum, nullable), socialChannelUrl (text, nullable), createdAt, updatedAt
       - circleMembers table: id, circleId (FK circles, onDelete cascade), contributorId (FK contributors, onDelete restrict), joinedAt (timestamp). Add unique constraint on (circleId, contributorId).
       - circleNotes table: id, circleId (FK circles, onDelete cascade), authorId (FK contributors, onDelete restrict), body (text, not null), createdAt, updatedAt
       - noteAttachments table: id, noteId (FK circleNotes, onDelete cascade), s3Key (text), fileName (text), mimeType (varchar 100), fileSizeBytes (integer), createdAt
       - circleResolutions table: id, circleId (FK circles, onDelete cascade, UNIQUE on circleId — one resolution per circle), submittedBy (FK contributors, onDelete restrict), problemSummary (text), recommendations (text), evidence (text), dissentingViews (text, nullable), implementationNotes (text, nullable), submittedAt, updatedAt
       - resolutionRatings table: id, resolutionId (FK circleResolutions, onDelete cascade, UNIQUE on resolutionId — one rating per resolution), raterId (FK contributors, onDelete restrict), rating (smallint, 1-5), feedback (text, nullable), createdAt

       - Creates GetObjectCommand with Bucket from env, Key = s3Key
       - Returns presigned URL via getSignedUrl

    2. Create packages/server/src/routes/circles.ts — a new Express Router following challenges.ts patterns:
       - Import authMiddleware, getDb, shared schemas, drizzle operators (eq, and, inArray, sql, desc)
       - Import generateUploadUrl and generateDownloadUrl from s3.service
       - Helper function requireCircleMember(circleId, contributorId, db) — returns boolean
       - Helper function requireChallenger(circleId, contributorId, db) — verifies contributor is the challenge creator for the circle's challenge, returns boolean

       ROUTES (declare in this order — static paths before parameterized):

       POST / — CM-only circle creation. Validate body with createCircleSchema. Check multi-circle limit for each memberIds entry: count active circles (status in forming/active/submitted) for each contributor, compare against their contributorProfiles.maxCircles (default 3). Use db.transaction: insert circle, insert circleMembers, set status to "active" if memberIds.length >= 3. Return 201 with created circle.

       GET / — List contributor's circles. Join circleMembers + circles + challenges to return CircleListItem[] for the authenticated contributor. Include memberCount via subquery.

       GET /:id — Circle workspace. Verify membership. Return circle details + challenge (title, brief, domain, skillsNeeded, type) + members (id, name from contributors) + socialChannel + socialChannelUrl.

       POST /:id/members — CM-only add member mid-challenge (CIRC-08). Validate contributorId in body. Check multi-circle limit. Insert circleMembers row. If circle was "forming" and now has >= 3 members, update status to "active".

       POST /:id/notes/attachment-url — DECLARE BEFORE /:id/notes/:noteId routes to avoid route shadowing. Verify membership. Validate body with attachmentUrlSchema. Generate S3 key: circle-notes/{circleId}/{uuid}/{timestamp}-{sanitizedFileName}. Call generateUploadUrl (adapt if needed to accept custom s3Key, or construct PutObjectCommand inline). Return { uploadUrl, s3Key }.

       GET /:id/notes — Verify membership. Paginated notes feed (cursor-based using createdAt desc, limit 20). For each note, include attachments (from noteAttachments). Return notes with author name (join contributors). Do NOT generate presigned download URLs here — frontend requests them lazily.

       POST /:id/notes — Verify membership. Validate body with postNoteSchema. Use db.transaction: insert circleNote, then insert noteAttachments rows for each attachment. Return 201 with created note + attachments.

       GET /:id/notes/:noteId/download/:attachmentId — Verify membership. Look up noteAttachment by id, verify it belongs to a note in this circle. Call generateDownloadUrl(attachment.s3Key). Return { downloadUrl, fileName }.

       PUT /:id/social — Verify membership. Validate body with setSocialChannelSchema. Update circle's socialChannel and socialChannelUrl. Return updated circle.

       POST /:id/resolution — Verify membership. Validate body with submitResolutionSchema. Insert circleResolution with submittedBy = contributor id. Catch unique violation (Postgres 23505) and return 409 if resolution already exists. Update circle status to "submitted". Return 201 with resolution.

       PUT /:id/resolution — Verify membership. Only allowed while circle status is "submitted" (not "completed"). Validate body with submitResolutionSchema. Update existing resolution in place. Return updated resolution.

       GET /:id/resolution — Verify membership OR verify challenger (challenge creator can view to rate). Return resolution + rating if exists.

       POST /:id/resolution/rating — Verify challenger identity (circle -> challenge -> createdBy must equal req.contributor.id). Validate body with rateResolutionSchema. Insert resolutionRating. Catch unique violation (23505) return 409. Update circle status to "completed". Return 201 with rating.
