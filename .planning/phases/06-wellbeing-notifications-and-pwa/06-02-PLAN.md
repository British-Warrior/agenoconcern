**7. Notification service** — Create packages/server/src/services/notification.service.ts:

```typescript
import webpush from 'web-push';
import { Resend } from 'resend';
```

- On module load: if VAPID_PUBLIC_KEY and VAPID_PRIVATE_KEY are non-empty, call `webpush.setVapidDetails(VAPID_SUBJECT, VAPID_PUBLIC_KEY, VAPID_PRIVATE_KEY)`. If empty, log warning and skip VAPID setup (graceful degradation).

- `async function notify(contributorId: string, payload: { type: NotificationType, title: string, body: string, url?: string })`:
  1. Insert into notifications table (always — this is the in-app record)
  2. Look up all pushSubscriptions for contributorId
  3. For each subscription, try `webpush.sendNotification(sub, JSON.stringify({ title, body, url }))`
     - On 410 or 404 status error: delete that push subscription row from DB
     - On other errors: log and continue
  4. If NO push subscriptions exist OR all push sends failed:
     - Look up contributor email from contributors table
     - If RESEND_API_KEY is non-empty and email exists, send email via Resend: `resend.emails.send({ from: 'Age No Concern <noreply@agenoconcern.org>', to: [email], subject: title, html: '<p>' + body + '</p>' + (url ? '<p><a href="' + CLIENT_URL + url + '">View</a></p>' : '') })`
     - If RESEND_API_KEY is empty, log warning (graceful degradation)

- `async function notifyBatch(contributorIds: string[], payload)`: calls notify for each (sequential to avoid rate limits).

**8. Server routes** — Create packages/server/src/routes/notifications.ts:

`GET /api/notifications` (auth required):
- Query notifications for contributorId, order by createdAt desc, limit 50
- Return array of Notification objects

`PATCH /api/notifications/:id/read` (auth required):
- Update readAt to now() where id = params.id AND contributorId = auth user
- Return 200

`POST /api/notifications/read-all` (auth required):
- Update readAt to now() for all unread notifications for contributorId
- Return 200

`POST /api/notifications/push-sub` (auth required):
- Parse body with pushSubscriptionSchema
- Upsert into pushSubscriptions: insert on conflict (endpoint) update p256dh, auth, updatedAt
- Return 201

`DELETE /api/notifications/push-sub` (auth required):
- Parse body.endpoint
- Delete from pushSubscriptions where endpoint and contributorId match
- Return 200

`GET /api/notifications/preferences` (auth required):
- Query contributorProfiles.notifyCircleActivity for contributorId
- Return { notifyCircleActivity } (default 'immediate' if null)

`PATCH /api/notifications/preferences` (auth required):
- Parse body with notificationPreferencesSchema
- Update contributorProfiles.notifyCircleActivity for contributorId
- Return 200
