  <files>

  <action>
    1. Create auth service (`packages/server/src/services/auth.service.ts`):
       - **Password hashing:** Use argon2 (Argon2id algorithm) for hashing passwords. `argon2.hash(password)` and `argon2.verify(hash, password)`.
       - **JWT tokens:** Use jose library. Create `createTokens(contributorId, role)` returning { accessToken, refreshToken }. Access token: 15-min expiry, HS256, payload { sub: contributorId, role }. Refresh token: 7-day expiry, HS256, payload { sub: contributorId, type: "refresh" }. JWT_SECRET from env.
       - **verifyToken(token):** Verify and decode JWT with jose.jwtVerify.
       - **register(name, email, password):** Validate with Zod schema from shared. Check email not taken. Hash password with argon2. Insert into contributors table with authProvider "email". Create JWT tokens. Return tokens + contributor.
       - **login(email, password):** Find contributor by email. Verify password with argon2.verify. Create JWT tokens. Return tokens + contributor.
       - **setAuthCookies(res, tokens):** Set access_token cookie (httpOnly, secure in prod, sameSite lax, maxAge 15min, path /). Set refresh_token cookie (httpOnly, secure in prod, sameSite lax, maxAge 7 days, path /api/auth/refresh).
       - **clearAuthCookies(res):** Clear both cookies with matching paths.

    2. Create auth routes (`packages/server/src/routes/auth.ts`):
       - `POST /api/auth/register`: Validate body with registerInput schema. Call authService.register. Set cookies. Return { contributor: { id, name, email, role } }.
       - `POST /api/auth/login`: Validate body with loginInput schema. Call authService.login. Set cookies. Return { contributor }.
       - `POST /api/auth/refresh`: Read refresh_token cookie. Verify it. Check type === "refresh". Look up contributor (verify still exists/active). Issue new access token only. Set new access_token cookie. Return { success: true }.
       - `POST /api/auth/logout`: Call clearAuthCookies. Return { message: "Logged out" }.
       - `GET /api/auth/me`: Use authMiddleware. Return { contributor } from req.contributor (look up full profile from DB).

    3. Mount auth routes in express-app.ts:
       - `app.use("/api/auth", authRoutes)`
       - Ensure cookie-parser and JSON body parser are mounted before auth routes

    4. Run database migration:
       - Run `drizzle-kit push` to push schema to PostgreSQL (or `drizzle-kit generate` + `drizzle-kit migrate` if DB exists)
       - If DATABASE_URL not set, the server should still start but log a warning. Auth endpoints will return 500 until DB is connected.
  </action>
  <verify>

    - `curl -X POST http://localhost:3000/api/auth/login -H "Content-Type: application/json" -d '{"email":"test@example.com","password":"SecurePass123!"}' -v` -- returns 200 with cookies

    1. Add Google OAuth to auth service and routes:
       - Initialise Arctic Google client: `new arctic.Google(GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, GOOGLE_REDIRECT_URI)`
       - `GET /api/auth/google`: Generate state + codeVerifier with arctic.generateState() and arctic.generateCodeVerifier(). Create auth URL with scopes ["openid", "profile", "email"]. Store state + codeVerifier in httpOnly cookies (sameSite lax, maxAge 10min). Redirect to Google.
       - `GET /api/auth/callback/google`: Read state + codeVerifier from cookies. Validate state matches query param. Call google.validateAuthorizationCode(code, codeVerifier). Decode ID token with arctic.decodeIdToken. Extract sub, name, email, picture. Find contributor by authProviderId OR email. If not found, create new contributor (authProvider "google", authProviderId = sub). Upsert oauthAccounts row with profile snapshot. Issue JWT, set auth cookies. Clear OAuth state cookies. Redirect to CLIENT_URL (e.g., http://localhost:5173/dashboard).
       - IMPORTANT: Google uses PKCE (codeVerifier), LinkedIn does NOT. Do not mix patterns.

    2. Add LinkedIn OAuth to auth service and routes:
       - Initialise Arctic LinkedIn client: `new arctic.LinkedIn(LINKEDIN_CLIENT_ID, LINKEDIN_CLIENT_SECRET, LINKEDIN_REDIRECT_URI)`
       - `GET /api/auth/linkedin`: Generate state with arctic.generateState(). Create auth URL with scopes ["openid", "profile", "email"]. NO codeVerifier for LinkedIn. Store state in httpOnly cookie. Redirect.
       - `GET /api/auth/callback/linkedin`: Read state from cookie. Validate. Call linkedin.validateAuthorizationCode(code). Decode ID token OR fetch userinfo endpoint. Extract sub, name, email, picture. Find or create contributor (same logic as Google). Store LinkedIn profile data as snapshot in oauthAccounts (LinkedIn tokens expire in 60 days -- do not rely on ongoing access). Issue JWT, set cookies, redirect.

    IMPORTANT: OAuth providers degrade gracefully. If GOOGLE_CLIENT_ID is not set, the /api/auth/google endpoint should return 501 "Google OAuth not configured" instead of crashing. Same for LinkedIn. This lets the server run in dev without all services configured.
