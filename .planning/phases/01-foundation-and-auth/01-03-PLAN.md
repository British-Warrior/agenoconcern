  <action>
    1. Create Tailwind v4 design system (`packages/web/src/styles/app.css`):
       - `@import "tailwindcss";`
       - `@theme { }` block with full colour palette from research Pattern 6:
         - Primary (deep navy): 900-500 in OKLCH, authority without corporate coldness
         - Accent (warm amber/gold): 600-400 in OKLCH, warmth not corporate
         - Neutral (warm greys): 950-50 in OKLCH
         - Semantic: success (green), warning (amber), error (red) -- all hitting 7:1 contrast on white
       - Typography: Inter font family, 18px base (1.125rem), never below 16px (1rem for --font-size-sm). Line height 1.6 normal, 1.75 relaxed.
       - Spacing: --spacing-touch: 3rem (48px minimum touch target)
       - IMPORTANT: This is Tailwind v4 CSS-first. NO tailwind.config.js file. NO @tailwind directives.
       - Verify all colour pairs hit 7:1+ contrast ratio (WCAG AAA). If unsure, increase lightness difference.

    2. Create UI primitive components (all in `packages/web/src/components/ui/`):
       - **Button.tsx:** Accepts variant (primary, secondary, outline, ghost), size (default, lg), disabled, loading, fullWidth. Renders as `<button>` with proper ARIA attributes. Minimum height 48px (touch target). Uses Tailwind classes from theme. Loading state shows spinner + "Loading..." text. All interactive states: hover, focus-visible (ring), active, disabled.
       - **Input.tsx:** Accepts label, error, helperText, type, required, id. Renders `<label>` + `<input>` pair with proper for/id linking. Large text (18px), generous padding. Error state with red border + error message. aria-invalid and aria-describedby when error present. Minimum height 48px.
       - **Card.tsx:** Simple container with padding, border, rounded corners, optional title. Semantic: uses `<section>` with optional `<h2>`.
       - **Alert.tsx:** Accepts variant (success, warning, error, info). Icon + text. Uses both colour AND icon to indicate type (never colour alone -- colour-blind safety). role="alert" for errors, role="status" for success/info.
       - **ConsentBanner.tsx:** GDPR cookie consent banner. Shows on first visit (check localStorage). Two EQUAL-weight buttons: "Accept All" and "Reject All" (ICO requirement -- reject must be as easy as accept). Optional "Manage Preferences" link expanding to show category toggles: Essential (always on, disabled toggle), Analytics (off by default). On accept/reject: store choice in localStorage, log consent record to `/api/auth/consent` if user is authenticated (or queue for after auth). Footer link "Manage Cookies" re-opens banner. Uses react-cookie-consent library for the banner shell, customised for equal buttons.

    3. Create layout components:
       - **AppShell.tsx:** `<header>` + `<main>` + `<footer>`. Header contains Navbar. Main renders `<Outlet />` (React Router). Footer has links to Privacy Policy, Cookie Policy, "Manage Cookies" button. Semantic HTML throughout. Full ARIA landmarks (banner, main, contentinfo).
       - **Navbar.tsx:** Logo/site name "Age No Concern" (text, not image). If authenticated: show contributor name + Logout button. If not: show Login link. Logout calls POST /api/auth/logout then redirects to landing. Logout button visible from ALL authenticated pages (AUTH-05 requirement).
       - **ProtectedRoute.tsx:** Wraps protected routes. Uses useAuth() hook. If loading, show spinner. If not authenticated, redirect to /login with return URL in query param. If authenticated, render children/Outlet.

    4. Create API client and auth hooks:
       - **`packages/web/src/lib/constants.ts`:** API_BASE_URL (empty string -- Vite proxy handles /api), route paths as constants.
       - **`packages/web/src/api/client.ts`:** Fetch wrapper. All requests include `credentials: "include"` (sends cookies). On 401 response: attempt token refresh by calling POST /api/auth/refresh. If refresh succeeds, retry original request. If refresh fails, redirect to login. JSON parsing with error handling.
       - **`packages/web/src/api/auth.ts`:** Functions: register(name, email, password), login(email, password), logout(), getMe(), refreshToken(), forgotPassword(email), resetPassword(token, newPassword), sendPhoneCode(phoneNumber), verifyPhoneCode(phoneNumber, code). Each calls the corresponding /api/auth/* endpoint via the fetch client.
       - **`packages/web/src/hooks/useAuth.ts`:** React context + hook. Uses TanStack Query to call getMe() on mount. Provides: contributor (or null), isAuthenticated, isLoading, login(), register(), logout(), error. QueryClient invalidates "me" query on login/register/logout.

    5. Set up React Router and QueryClient:
       - **main.tsx:** Wrap App in QueryClientProvider.
       - **App.tsx:** BrowserRouter with routes. Public routes: / (Landing), /login, /register, /phone-login, /forgot-password, /reset-password, /privacy, /cookies. Protected routes (wrapped in ProtectedRoute): /dashboard. AppShell wraps all routes. ConsentBanner rendered globally (inside App, outside routes).
