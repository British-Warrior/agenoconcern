  <action>
    Create `useFocusTrap.ts` in `components/a11y/`. Follow the exact pattern from ARCHITECTURE.md Pattern 3:
    - Export `useFocusTrap(active: boolean)` returning a `containerRef`
    - FOCUSABLE_SELECTORS: `a[href]`, `button:not([disabled])`, `input:not([disabled])`, `select:not([disabled])`, `textarea:not([disabled])`, `[tabindex]:not([tabindex="-1"])`
    - When `active` becomes true: capture `document.activeElement` as trigger, find all focusable children, focus the first one
    - Intercept Tab/Shift-Tab keydown: if on last element and Tab, wrap to first; if on first and Shift-Tab, wrap to last
    - On cleanup (active becomes false or unmount): restore focus to captured trigger element
    - Type the ref as `useRef<HTMLElement>(null)` so Modal can cast to `RefObject<HTMLDivElement>`

    Create `Modal.tsx` in `components/ui/`. Follow ARCHITECTURE.md Pattern 4:
    - Props: `isOpen: boolean`, `onClose: () => void`, `titleId: string`, `children: ReactNode`, `className?: string`
    - Use `useFocusTrap(isOpen)` — attach returned ref to the outer dialog div
    - Render with `createPortal` to `document.body` for proper stacking
    - Outer div: `role="dialog"`, `aria-modal="true"`, `aria-labelledby={titleId}`
    - Backdrop: `fixed inset-0 bg-black/50 backdrop-blur-sm`, `onClick={onClose}`, `aria-hidden="true"`
    - Panel: `relative bg-white rounded-[var(--radius-lg)] shadow-xl w-full max-w-md p-6`
    - Escape key: `useEffect` adds keydown listener when `isOpen`, calls `onClose` on Escape
    - Return `null` when `!isOpen`

    Use project conventions: named exports, no default exports, Tailwind for styling, CSS custom properties for design tokens (--radius-lg, etc).
