The page has two sections controlled by a tab/toggle at the top:
1. **Active Flags** (default view) — uses `useAttentionFlags()` hook
2. **Signal History** — uses `useAttentionHistory()` hook

**Active Flags section:**
- Loading state: "Loading..." (same pattern as InstitutionManagement)
- Empty state: "No flagged contributors at your institution." (neutral-500 text)
- Error state: "Failed to load attention flags." with error message
- Each flag rendered as a card showing: contributor name (bold), signal type (badge/tag), cohort size and flagged count (if not null), created date (formatted), and a "Resolve" button
- "Resolve" button opens a confirm dialog (local state pattern from InstitutionManagement.tsx ConfirmDialog)
- Confirm dialog contains: flag summary text, a `<textarea>` for follow-up notes (required, placeholder "Enter follow-up notes..."), "Cancel" and "Confirm Resolve" buttons
- "Confirm Resolve" button is DISABLED when `followUpNotes.trim() === ""` (frontend validation matching server Zod schema)
- On confirm: call `resolveFlag.mutateAsync({ flagId, followUpNotes })`, close dialog, clear notes state
- While mutation is pending, show loading state on "Confirm Resolve" button

**Signal History section:**
- Uses `useAttentionHistory()` hook
- Renders all flags (resolved and unresolved) in a table or card list ordered by date
- Each entry shows: contributor name, signal type, created date, status (resolved/unresolved), and for resolved flags: cleared date and follow-up notes (truncated with full text on hover/expand)

**Tab/toggle UI:**
- Two buttons at the top: "Active Flags" and "History"
- Active tab has primary styling, inactive has ghost/outline styling
- Use local state (`useState<"active" | "history">("active")`) to switch views
