**Route 1: GET /attention (active flags)**
1. Extract `cmId` from `req.contributor!.id` (non-null assertion, matches existing admin.ts pattern)
2. Look up CM's institution: query `contributorInstitutions` where `contributorId = cmId`, LIMIT 1. If no assignment, return 403 `{ error: "No institution assigned to this community manager" }`
3. Query `ithinkAttentionFlags` INNER JOIN `contributors` (for contributor name) WHERE `institutionId = assignment.institutionId` AND `clearedAt IS NULL`, ORDER BY `createdAt DESC`
4. Select fields: `id, contributorId, contributorName (from contributors.name), signalType, cohortSize, flaggedCount, createdAt`
5. Return the array as JSON

**Route 2: GET /attention/history (all flags including resolved)**
1. Same institution lookup as Route 1
2. Query `ithinkAttentionFlags` INNER JOIN `contributors` WHERE `institutionId = assignment.institutionId` (no clearedAt filter), ORDER BY `createdAt DESC`
3. Select fields: same as Route 1 plus `clearedAt, followUpNotes, clearedBy`
4. Return the array as JSON

IMPORTANT: Register `/attention/history` BEFORE any `/attention/:flagId` route to avoid Express matching "history" as a flagId parameter. Route order within admin.ts matters.

LIMIT 1 assumption for CM institution: pilot-scale assumption (one CM manages one institution). Documented as known limitation.

**Inline Zod schema** (at the top of admin.ts, after imports):
```typescript
const resolveAttentionFlagSchema = z.object({
  followUpNotes: z.string().min(1, "Follow-up notes are required"),
});
```
Add `z` import from `zod` if not already imported (check existing imports first).

**Route: POST /attention/:flagId/resolve**
1. Validate `flagId` against `UUID_PATTERN` (already defined in admin.ts). Return 400 if invalid.
2. Parse `req.body` with `resolveAttentionFlagSchema.safeParse()`. Return 400 with `result.error.flatten()` if invalid.
3. Look up CM's institution (same `contributorInstitutions` LIMIT 1 pattern). Return 403 if no assignment.
4. Fetch the flag: query `ithinkAttentionFlags` WHERE `id = flagId` AND `institutionId = assignment.institutionId`, LIMIT 1. Return 404 `{ error: "Flag not found" }` if not found (same 404 whether flag doesn't exist or belongs to different institution — prevents enumeration).
5. Check `flag.clearedAt` is not null. Return 409 `{ error: "Flag already resolved" }` if already cleared.
6. UPDATE `ithinkAttentionFlags` SET `clearedBy = cmId, clearedAt = new Date(), followUpNotes = result.data.followUpNotes, updatedAt = new Date()` WHERE `id = flagId`. Use `.returning()`.
7. Return the updated flag object as JSON.

Security: The flag must belong to the CM's institution (step 4 checks both `id` AND `institutionId`). A CM cannot resolve flags from other institutions.
