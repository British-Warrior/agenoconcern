Add a companion data table to AttentionTrendChart, controlled by a toggle button:

1. Import `useState` from React (add to existing import).

2. Inside the `AttentionTrendChart` component (only in the branch where `data.length > 0`), add state: `const [showTable, setShowTable] = useState(false);`

3. After the chart's closing `</div>` (the `h-56` container), add:

   a. A toggle button:
   ```tsx
   <button
     type="button"
     onClick={() => setShowTable((v) => !v)}
     aria-expanded={showTable}
     className="text-sm text-primary-700 hover:underline mt-2 focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-accent-500 rounded px-1"
   >
     {showTable ? "Hide data table" : "View as table"}
   </button>
   ```

   b. The companion table (conditionally rendered when `showTable` is true):
   ```tsx
   {showTable && (
     <table className="mt-3 w-full text-sm border-collapse">
       <caption className="sr-only">Attention flags by week</caption>
       <thead>
         <tr>
           <th scope="col" className="text-left py-1 pr-4 font-medium text-neutral-700 border-b border-neutral-200">Week</th>
           <th scope="col" className="text-right py-1 font-medium text-neutral-700 border-b border-neutral-200">Flags</th>
         </tr>
       </thead>
       <tbody>
         {data.map((point) => (
           <tr key={point.isoWeek}>
             <td className="py-1 pr-4 text-neutral-600">{formatWeekLabel(point.isoWeek)}</td>
             <td className="py-1 text-right text-neutral-900">{point.count}</td>
           </tr>
         ))}
       </tbody>
     </table>
   )}
   ```

4. Use the existing `formatWeekLabel` helper (already defined in the file) for week labels — same format as the chart axis.

5. Use the original `data` prop (TrendPoint[]) for the table rows, NOT `chartData` (which adds a `label` field for the chart). Both work, but `data` is the canonical source.

6. Do NOT add `accessibilityLayer={false}` to the BarChart — recharts 3.x has it enabled by default and it adds keyboard nav between bars, which complements the table.

7. The table caption is `sr-only` so screen readers announce context. The table itself is visible when toggled, allowing low-vision users to zoom/scroll it.
