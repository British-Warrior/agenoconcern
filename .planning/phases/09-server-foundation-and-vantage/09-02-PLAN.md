    2. Create `packages/server/src/middleware/api-key-auth.ts` with:

       a. **Declaration merging** for `Express.Request` to add `vantageClient`:
          ```typescript
          declare global {
            namespace Express {
              interface Request {
                vantageClient?: {
                  keyId: string;
                  scopes: string[];
                };
              }
            }
          }
          ```

       b. **`apiKeyMiddleware` function:**
          - Read `X-API-Key` from `req.headers["x-api-key"]`
          - If missing or not a string, return 401 `{ error: "API key required" }`
          - SHA-256 hash the incoming key: `createHash("sha256").update(raw).digest("hex")`
          - Query `api_keys` table: `WHERE key_hash = $incomingHash AND is_active = true AND expires_at > now()`
          - Use `eq()`, `and()`, `gt()` from drizzle-orm, `.limit(1)`
          - If no matching key found, return 401 `{ error: "Invalid or expired API key" }`
          - **Timing-safe comparison:** After finding a row by hash, do a secondary `timingSafeEqual` check comparing `Buffer.from(incomingHash, "hex")` with `Buffer.from(key.keyHash, "hex")` to prevent timing attacks on the DB lookup. If lengths differ or comparison fails, return 401.
          - Update `last_used_at` to `new Date()` on the matched key (fire-and-forget, no await needed)
          - Set `req.vantageClient = { keyId: key.id, scopes: key.scopes as string[] }`
          - Call `next()`

       c. **`vantageRateLimiter` constant** using `rateLimit()` from express-rate-limit:
          ```typescript
          import { rateLimit } from "express-rate-limit";

          export const vantageRateLimiter = rateLimit({
            windowMs: 60 * 1000,
            limit: 60,
            standardHeaders: "draft-8",
            legacyHeaders: false,
            keyGenerator: (req) => {
              const apiKey = req.headers["x-api-key"];
              return typeof apiKey === "string" ? apiKey : (req.ip ?? "unknown");
            },
            handler: (_req, res) => {
              res.status(429).json({ error: "Rate limit exceeded" });
            },
          });
          ```

       Imports needed: `createHash`, `timingSafeEqual` from `"node:crypto"`, `Request`/`Response`/`NextFunction` from `"express"`, `eq`/`and`/`gt` from `"drizzle-orm"`, `getDb` from `"../db/index.js"`, `apiKeys` from `"../db/schema.js"`, `rateLimit` from `"express-rate-limit"`.

       IMPORTANT: Never log the raw API key value. Do not import or use `authMiddleware` — this is a completely separate auth path.
  </action>

    1. Create `packages/server/src/routes/vantage.ts`:

       a. Import `Router` from express, `apiKeyMiddleware`, `vantageRateLimiter` from `"../middleware/api-key-auth.js"`, `getDb` from `"../db/index.js"`, relevant schema tables, `authMiddleware` and `requireRole` from `"../middleware/auth.js"` (for the admin-only key creation endpoint), `randomBytes` and `createHash` from `"node:crypto"`.

       b. Create the router: `const router = Router();`

       c. **POST /keys** (admin-only, cookie-auth — for creating API keys):
          - Apply `authMiddleware` and `requireRole("admin")` on this route only (NOT apiKeyMiddleware)
          - Accept body: `{ name: string, scopes?: string[], expiresInDays?: number }`
          - Validate with zod: name required (1-100 chars), scopes defaults to `["vantage"]`, expiresInDays defaults to 365
          - Generate raw key: `randomBytes(32).toString("hex")`
          - Hash it: `createHash("sha256").update(raw).digest("hex")`
          - Insert into `api_keys` table: name, keyHash, scopes, expiresAt (now + expiresInDays), createdBy (req.contributor.id)
          - Return 201: `{ key: raw, id: inserted.id, name, scopes, expiresAt }` — this is the ONLY time the raw key is returned
          - Never log the raw key variable

       d. **Guarded routes** (API-key auth + rate limit):
          Apply `vantageRateLimiter` then `apiKeyMiddleware` to remaining routes. CRITICAL: rate limiter MUST come before apiKeyMiddleware (throttle unauthenticated flooding).

          ```typescript
          // Rate limit first, then auth
          router.use("/challenges", vantageRateLimiter, apiKeyMiddleware);
          ```

       e. **GET /challenges** (VANTAGE-authenticated):
          - Query `challenges` table where `status = 'open'`
          - Support query params: `page` (default 1), `limit` (default 20, max 100), `domain` (optional filter)
          - Return 200: `{ challenges: Array<{ id, title, description, domain, skillsNeeded, type, deadline, circleSize, status, createdAt }>, total: number, page: number, limit: number }`

       f. **GET /challenges/:id** (VANTAGE-authenticated):
          - Query single challenge by UUID
          - Return 200 with full challenge object, or 404 `{ error: "Challenge not found" }`

    2. Update `packages/server/src/express-app.ts`:
       - Import `vantageRoutes` from `"./routes/vantage.js"`
       - Add before the error handler: `app.use("/api/vantage", vantageRoutes);`

    3. Create `packages/server/VANTAGE-CONTRACT.md` with:

       ```markdown
       # VANTAGE API Contract

       Version: 1.0
       Last updated: 2026-03-16

       ## Authentication

       All VANTAGE endpoints (except key creation) require:
       - Header: `X-API-Key: <raw-api-key>`

       Key creation requires admin cookie authentication (JWT).

       ## Rate Limits

       - 60 requests per minute per API key
       - Rate limit headers: `RateLimit` (draft-8 standard)
       - Exceeding limit: 429 `{ error: "Rate limit exceeded" }`

       ## Endpoints

       ### POST /api/vantage/keys
       **Purpose:** Create a new API key (admin only)
       **Auth:** JWT cookie (admin role)
       **Request body:**
       ```json
       {
         "name": "string (required, 1-100 chars)",
         "scopes": ["string"] (optional, default: ["vantage"]),
         "expiresInDays": number (optional, default: 365)
       }
       ```
       **Response 201:**
       ```json
       {
         "key": "string (64-char hex — shown once, never retrievable again)",
         "id": "uuid",
         "name": "string",
         "scopes": ["string"],
         "expiresAt": "ISO 8601 timestamp"
       }
       ```
       **Response 401:** `{ "error": "Authentication required" }`
       **Response 403:** `{ "error": "Insufficient permissions" }`

       ### GET /api/vantage/challenges
       **Purpose:** List open challenges
       **Auth:** X-API-Key header
       **Query params:**
       - `page` (number, default: 1)
       - `limit` (number, default: 20, max: 100)
       - `domain` (string, optional — filter by domain)
       **Response 200:**
       ```json
       {
         "challenges": [
           {
             "id": "uuid",
             "title": "string",
             "description": "string",
             "domain": ["string"],
             "skillsNeeded": ["string"],
             "type": "paid | free",
             "deadline": "date | null",
             "circleSize": number,
             "status": "open",
             "createdAt": "ISO 8601 timestamp"
           }
         ],
         "total": number,
         "page": number,
         "limit": number
       }
       ```
       **Response 401:** `{ "error": "API key required" }` or `{ "error": "Invalid or expired API key" }`
       **Response 429:** `{ "error": "Rate limit exceeded" }`

       ### GET /api/vantage/challenges/:id
       **Purpose:** Get a single challenge by ID
       **Auth:** X-API-Key header
       **Response 200:** Single challenge object (same shape as list item)
       **Response 401:** `{ "error": "API key required" }` or `{ "error": "Invalid or expired API key" }`
       **Response 404:** `{ "error": "Challenge not found" }`
       **Response 429:** `{ "error": "Rate limit exceeded" }`
       ```

    IMPORTANT: The key creation endpoint uses cookie auth (authMiddleware + requireRole), NOT API key auth. This is intentional — you need an admin session to create keys. The challenge endpoints use API key auth.
