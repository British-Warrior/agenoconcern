  <action>
    1. **API client additions** (`packages/web/src/api/challenges.ts`):
       - `createChallenge(data)` -> POST /api/challenges
       - `updateChallenge(id, data)` -> PUT /api/challenges/:id
       - `getTeamSuggestions(challengeId)` -> GET /api/challenges/:id/team-suggestions
       - `getMyChallenges()` -> GET /api/challenges?createdBy=me (or a dedicated CM endpoint; if the feed endpoint doesn't support this filter, add a query param `mine=true` that filters by createdBy on the server — check 03-01-SUMMARY.md for what was implemented and add the param if missing)

    2. **Hook additions** (`packages/web/src/hooks/useChallenges.ts`):
       - `useCreateChallenge()`: useMutation calling createChallenge. On success, invalidate `["challenges", "feed"]`.
       - `useUpdateChallenge(challengeId)`: useMutation calling updateChallenge. On success, invalidate feed and challenge detail.
       - `useTeamSuggestions(challengeId)`: useQuery calling getTeamSuggestions. Enabled only when challengeId is provided.
       - `useMyChallenges()`: useQuery fetching CM's own challenges.

    3. **ChallengeForm** (`packages/web/src/components/challenges/ChallengeForm.tsx`):
       - Structured form per user decision: title, description, domain, skills needed, type (paid/free), deadline, Circle size.
       - **Title:** text input, 5-200 chars.
       - **Description:** textarea, 20-5000 chars. Include a "Brief excerpt" field (10-500 chars) — this is the short text shown in expanded challenge header.
       - **Domain:** select dropdown from DOMAIN_TAXONOMY constant (import from @indomitable-unity/shared).
       - **Skills needed:** tag input — text input where CM types a skill and presses Enter to add it as a pill/chip. Each chip has an X to remove. Max 20 skills.
       - **Type:** radio group: Paid / Free.
       - **Deadline:** date input (optional). Use native HTML date input.
       - **Circle size:** number input, min 2 max 10, default 4.
       - Client-side validation matching createChallengeSchema from shared.
       - Props: onSubmit callback, optional initialValues (for edit mode), isEditing boolean.
       - When isEditing and challenge has interests (interestCount > 0), show all fields as disabled/read-only with message: "Challenge cannot be edited after interest has been expressed."
       - Submit button: "Post Challenge" (create) or "Save Changes" (edit).

    4. **ManageChallengeRow** (`packages/web/src/components/challenges/ManageChallengeRow.tsx`):
       - A row for each challenge the CM has posted, showing: title, status badge, interest count, created date.
       - **Actions based on state:**
         - Status "open" + interestCount === 0: "Edit" button (opens ChallengeForm in edit mode), "Close" button.
         - Status "open" + interestCount > 0: "View Interests" button (expands to show interested names), "View Team Suggestions" button, "Close" button. Edit disabled.
         - Status "closed"/"archived": Read-only display.
       - Expandable to show interested contributor names (uses getInterests endpoint — CM sees all names per conditional visibility rules).

    5. **TeamCompositionCard** (`packages/web/src/components/challenges/TeamCompositionCard.tsx`):
       - Displays one suggested team composition.
       - Props: composition (TeamComposition type from shared).
       - Shows: ranked label ("Composition 1: Best Match" / "Composition 2: Most Diverse" / "Composition 3: Balanced"), list of contributor names with their skills as tag pills, coverageScore as "X% skills covered", diversityScore, balanceScore as progress bars or numeric displays.
       - CM can click "Select this team" — this is informational for Phase 3 (actual Circle formation is Phase 4). For now, show selection as a visual highlight/checkmark. Store selection in local state only.

    6. **ChallengeManage page** (`packages/web/src/pages/challenges/ChallengeManage.tsx`):
       - List of CM's posted challenges using ManageChallengeRow.
       - "Post a Challenge" button at top that shows ChallengeForm (inline expand or modal — use inline expand for simplicity).
       - When a challenge row's "View Team Suggestions" is clicked, fetch and display TeamCompositionCards below the row.
       - Empty state: "You haven't posted any challenges yet."
