  <action>
    First install Radix dependencies: `cd packages/web && npm install @radix-ui/react-accordion @radix-ui/react-collapsible`

    1. **API client** (`packages/web/src/api/challenges.ts`):
       - Use the project's existing fetch wrapper (check `packages/web/src/lib/fetch.ts` or equivalent).
       - `getFeed(params: { domain?, type?, page? })` -> GET /api/challenges with query params. Returns `{ challenges, page, hasMore }`.
       - `getChallenge(id)` -> GET /api/challenges/:id
       - `toggleInterest(challengeId, note?)` -> POST /api/challenges/:id/interest with { note } body
       - `getInterests(challengeId)` -> GET /api/challenges/:id/interests
       - Import Challenge and related types from `@agenoconcern/shared`.

    2. **Hooks** (`packages/web/src/hooks/useChallenges.ts`):
       - `useChallengeFeed(filters)`: TanStack Query v5 `useInfiniteQuery` with queryKey `["challenges", "feed", filters]`, initialPageParam: 1, getNextPageParam returns `lastPage.hasMore ? lastPage.page + 1 : undefined`. Per RESEARCH Pattern 7.
       - `useInterestToggle(challengeId)`: `useMutation` calling toggleInterest. On success, invalidate `["challenges", "feed"]` and `["challenges", challengeId, "interests"]` query keys.
       - `useChallengeInterests(challengeId)`: `useQuery` for GET /:id/interests. Only enabled when challengeId is provided.
       - Export `ChallengeFilters` type: `{ domain?: string; type?: 'paid' | 'free' }`.

    3. **ChallengeSubSection** (`packages/web/src/components/challenges/ChallengeSubSection.tsx`):
       - Wrap Radix `Collapsible.Root` with `defaultOpen={true}` (sub-sections open by default so user sees all content).
       - Props: label (string), defaultOpen (boolean, default true), children.
       - Trigger: full-width row with label text and a chevron icon that rotates via `data-[state=open]:rotate-180`.
       - Content area: padding for readable content.
       - Border-top separator between sub-sections.
       - Per RESEARCH Pattern 2.

    4. **ChallengeRow** (`packages/web/src/components/challenges/ChallengeRow.tsx`):
       - Single Accordion.Item component. Props: challenge object (from API).
       - **Collapsed state (Accordion.Trigger):** Show challenge title (left) and type badge (right). Type badge: "Paid" with accent colour, "Free" with neutral colour. Use pill-shaped badge (rounded-full, px-2, py-0.5, text-sm).
       - **Expanded state (Accordion.Content):** Render 4 ChallengeSubSection components:
         (a) "Description" — challenge.description text and challenge.brief
         (b) "Skills & Domain" — domain as a tag, skillsNeeded as a list of tag pills
         (c) "Deadline & Timeline" — formatted deadline date, or "No deadline" if null
         (d) "Interest" — interest count display + InterestButton component
       - Animate expand/collapse using Tailwind CSS keyframes: `data-[state=open]:animate-slideDown` / `data-[state=closed]:animate-slideUp`. Define these keyframes in the Tailwind CSS theme (in `packages/web/src/styles/` or equivalent). Use `overflow-hidden` on Accordion.Content.

    5. **ChallengeAccordion** (`packages/web/src/components/challenges/ChallengeAccordion.tsx`):
       - Radix `Accordion.Root` with `type="multiple"` (multiple challenges can be open at once).
       - Maps over challenges array rendering ChallengeRow for each.
       - Key each Accordion.Item by challenge.id (stable ID, not array index — per RESEARCH Pitfall 4).
       - Flex column with gap-2.

    6. **FilterBar** (`packages/web/src/components/challenges/FilterBar.tsx`):
       - Horizontal row of filter chips (per RESEARCH Pattern 8 recommendation — 3 filter dimensions fit a chip bar, not a drawer).
       - **Domain filter:** Dropdown/select from DOMAIN_TAXONOMY constant (imported from @agenoconcern/shared). Show "All Domains" as default.
       - **Type filter:** Three pill chips: All / Paid / Free. Active chip gets accent-600 background.
       - **Timeline filter:** Three pill chips: Any / This Week / This Month. "This Week" filters to deadline within 7 days, "This Month" within 30 days. These are computed client-side or passed as query params.
       - "Clear all" text link appears only when any filter is active.
       - On mobile: horizontal scroll (overflow-x-auto, flex-nowrap) rather than wrapping.
       - Props: filters state object, onFiltersChange callback.

    7. **ChallengeFeed page** (`packages/web/src/pages/challenges/ChallengeFeed.tsx`):
       - Page layout: heading "Challenges", FilterBar, then ChallengeAccordion with the feed data.
       - Use `useChallengeFeed(filters)` with filter state managed in component.
       - Flatten pages: `data.pages.flatMap(p => p.challenges)`.
       - **Infinite scroll sentinel:** A `<div ref={sentinelRef}>` at the bottom. Use IntersectionObserver (browser native, no library) to call `fetchNextPage()` when sentinel enters viewport with 200px rootMargin. Per RESEARCH Pattern 7 / Intersection Observer example.
       - Show loading spinner during initial load and while fetching next page.
       - **Empty state:** If no challenges match filters, show a friendly message: "No challenges match your filters" with a "Clear filters" action. If no challenges at all, show "No challenges posted yet — check back soon."
       - **Recommendations:** Per user decision, match scores are internal and used for feed ordering. The feed is already ordered by relevance. Do NOT add a separate "Recommended for you" section — the relevance-ordering IS the recommendation (per Claude's discretion choice: inline ordering over separate section, which is simpler and avoids duplicating challenges).

    8. **App.tsx route:** Add `/challenges` route inside ProtectedRoute, importing ChallengeFeed page.

    **Tailwind animation keyframes:** Add slideDown/slideUp keyframes for accordion animation. Check if `packages/web/src/styles/` has a main CSS file with @theme block (Tailwind v4 CSS-first config). Add:
    ```css

      from { height: 0; }
      to { height: var(--radix-accordion-content-height); }
    }
    @keyframes slideUp {
      from { height: var(--radix-accordion-content-height); }
      to { height: 0; }
    }
    ```
    And in the @theme block: `--animate-slideDown: slideDown 200ms ease-out; --animate-slideUp: slideUp 200ms ease-out;`

  <action>
    Create `InterestButton.tsx` — the interest expression component rendered inside the "Interest" sub-section of each expanded challenge row.

    **Props:** challengeId (string), myInterest (the current contributor's interest status from the feed response: `{ status: 'active', note: string | null } | null`), interestCount (number).

    **States:**
    - **No interest yet:** Show "I'm Interested" button (primary style, accent colour). On click, show a small inline form/modal with an optional note textarea (max 500 chars, placeholder "Why are you interested? (optional)") and "Express Interest" / "Skip note" buttons. Calling useInterestToggle mutation with optional note.
    - **Active interest:** Show "Interested" button in active/checked state (filled accent, checkmark icon or similar). Below it show the note if one exists. Show "Withdraw" text button (subtle, secondary style).
    - **Withdrawn (cooldown active):** Show "Withdrawn" state. If the API returned 429 with cooldownRemainingHours, display "You can re-express interest in X hours." Disable the interest button during cooldown.
    - **Loading:** Disable button and show spinner during mutation.

    **Soft capacity warning:** After successful interest expression, if the response includes `activeInterestCount >= maxCircles`, show an inline warning: "You're interested in {N} challenges — sure you have capacity?" Style as a subtle amber/warning banner below the button. This is a nudge only — does NOT block the action (per user decision: no hard limit).

    **Interest count display:** Show "{N} interested" text near the button. Update optimistically or via query invalidation after toggle.

    **Error handling:** If toggle fails (network error, 429 cooldown), show error via toast or inline alert. Do not crash.
