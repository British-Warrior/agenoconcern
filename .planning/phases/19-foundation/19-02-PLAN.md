  <action>
    **Install dependencies:**
    Run from the monorepo root:
    ```
    cd packages/web && pnpm add -D eslint eslint-plugin-jsx-a11y @eslint/js typescript-eslint
    ```

    **Create `packages/web/eslint.config.js`** (ESLint flat config format — ESLint 9+):
    ```js
    import js from "@eslint/js";
    import tseslint from "typescript-eslint";
    import jsxA11y from "eslint-plugin-jsx-a11y";

    export default [
      js.configs.recommended,
      ...tseslint.configs.recommended,
      jsxA11y.flatConfigs.strict,
      {
        files: ["src/**/*.{ts,tsx}"],
        languageOptions: {
          parserOptions: {
            ecmaFeatures: { jsx: true },
          },
        },
      },
      {
        ignores: ["dist/**", "node_modules/**"],
      },
    ];
    ```

    Key points:
    - Use `jsxA11y.flatConfigs.strict` (NOT `.recommended`) — strict mode is the requirement (TOOL-01).
    - Include typescript-eslint for proper TS parsing.
    - Only lint `src/**/*.{ts,tsx}` — skip dist and node_modules.

    **Add lint script to package.json:**
    Add to the `"scripts"` section:
    ```
    "lint": "eslint src/"
    ```
  </action>

    `cd packages/web && pnpm exec eslint --version` prints ESLint 9.x. `cat packages/web/eslint.config.js` confirms jsx-a11y strict config exists.
  </verify>

<task type="auto">
  <name>Task 2: Run lint, fix all existing violations, verify clean pass</name>
  <files>
    packages/web/eslint.config.js
  </files>
  <action>
    **Run the lint:**
    ```
    cd packages/web && pnpm run lint
    ```

    **Fix ALL reported violations.** Common jsx-a11y strict violations to expect:
    - `jsx-a11y/click-events-have-key-events` — elements with onClick need onKeyDown/onKeyUp. Fix by using `<button>` instead of `<div onClick>` or adding keyboard handlers.
    - `jsx-a11y/no-static-element-interactions` — non-interactive elements with event handlers. Fix by using semantic elements.
    - `jsx-a11y/label-has-associated-control` — labels not linked to inputs. Fix by adding htmlFor or wrapping.
    - `jsx-a11y/img-redundant-alt` — alt text containing "image" or "picture". Fix alt text.
    - `jsx-a11y/no-noninteractive-element-interactions` — e.g., `<li onClick>`. Fix by using button inside li.

    For each violation:
    1. Read the file and understand the context
    2. Fix using the semantically correct approach (prefer semantic HTML over ARIA overrides)
    3. Do NOT use eslint-disable comments unless the rule is genuinely a false positive (document why in a comment if so)

    **If the eslint config itself causes unexpected parser errors** (e.g., on .js config files being linted), add them to the ignores array.

    **After fixes, re-run:**
    ```
    cd packages/web && pnpm run lint
    ```
    Must exit 0 with no errors.

If 19-01 has NOT run yet (parallel wave), lint checks the current Link-based Navbar — fix any violations found in the current code. Both plans are wave 1 so either execution order is valid.
  </action>
