    1. **Migration `0006_institution_portal_accounts.sql`:** Create `institution_portal_accounts` table with columns: `id` UUID PK default random, `institution_id` UUID NOT NULL FK -> institutions(id) ON DELETE CASCADE, `email` VARCHAR(255) NOT NULL UNIQUE, `password_hash` TEXT NOT NULL, `created_by` UUID NOT NULL FK -> contributors(id) ON DELETE RESTRICT, `is_active` BOOLEAN NOT NULL DEFAULT true, `created_at` TIMESTAMPTZ NOT NULL DEFAULT now(), `updated_at` TIMESTAMPTZ NOT NULL DEFAULT now(). Add a unique partial index: `CREATE UNIQUE INDEX institution_portal_accounts_active_institution ON institution_portal_accounts (institution_id) WHERE is_active = true;` — this enforces one active portal account per institution while allowing deactivated rows.

    2. **Schema (`schema.ts`):** Add `institutionPortalAccounts` table definition at the end of the file, matching the migration. Use the same patterns as existing tables (uuid PK, references, timestamps).

    3. **Service (`portal-auth.service.ts`):** Create in `packages/server/src/services/`. Model directly on `auth.service.ts`.
       - `createPortalTokens(portalAccountId, institutionId)` — returns `{ accessToken, refreshToken }`. Access token JWT payload: `{ sub: portalAccountId, institutionId, type: "portal" }`, expiry 15m. Refresh token JWT payload: `{ sub: portalAccountId, type: "portal_refresh" }`, expiry 7d. Use `jose.SignJWT` with HS256 and same `getEnv().JWT_SECRET`.
       - `setPortalAuthCookies(res, tokens)` — sets `portal_access_token` cookie (httpOnly, sameSite lax, path "/", maxAge 15min) and `portal_refresh_token` cookie (httpOnly, sameSite lax, path "/api/portal/refresh", maxAge 7d). Use `getEnv().NODE_ENV === "production"` for secure flag.
       - `clearPortalAuthCookies(res)` — clears both portal cookies with matching paths.
       - `portalLogin(email, password)` — looks up `institutionPortalAccounts` by email (lowercased) where `isActive = true`, verifies password with `argon2.verify`, returns `{ tokens, account: { id, institutionId } }`. Throws `PortalAuthError` on failure.
       - `PortalAuthError` class extending `Error` with `statusCode`.

    4. **Middleware (`portal-auth.ts`):** Create in `packages/server/src/middleware/`. Mirror `auth.ts` structure exactly.
       - Reads `portal_access_token` cookie, verifies with `jose.jwtVerify`, checks `payload.type === "portal"` (rejects contributor tokens), attaches `req.portalUser = { id: payload.sub, institutionId: payload.institutionId }`.
       - Augment Express Request type with `portalUser?: { id: string; institutionId: string }`.
       - Returns 401 JSON on missing/invalid/wrong-type token.

    5. **Routes (`portal-auth.ts` in `routes/portal/`):** Create Express Router.
       - `POST /login` — validates `{ email, password }` with zod, calls `portalLogin()`, sets cookies, returns `{ ok: true }`.
       - `POST /logout` — clears portal cookies, returns `{ ok: true }`. No auth required.
       - `POST /refresh` — reads `portal_refresh_token` cookie, verifies JWT, checks `type === "portal_refresh"`, loads account from DB (re-reads institutionId for freshness per research pitfall 4), creates new tokens, sets cookies, returns `{ ok: true }`. Returns 401 if token invalid or account deactivated.
       - `GET /me` — protected by `portalAuthMiddleware`, loads account from DB, returns `{ id, email, institutionId, institutionName }` (join institutions table for name).
       - `POST /admin/create-portal-account` — protected by `authMiddleware` + `requireRole("community_manager")` (NOT portal auth — CMs are contributors). Validates `{ institutionId, email, password? }` with zod. If no password provided, generate random 16-char base64url password. Hash with argon2. Check no existing active account for institution (409 if exists). Insert into DB. Return `{ id, email, password }` (plaintext shown once).
       - `PATCH /admin/:accountId/active` — protected by `authMiddleware` + `requireRole("community_manager")`. Validates `{ isActive: boolean }`. Updates `is_active` and `updated_at`. Returns updated account.
       - `POST /admin/:accountId/reset-password` — protected by `authMiddleware` + `requireRole("community_manager")`. Generates new random password, hashes with argon2, updates account. Returns `{ password }` (plaintext shown once).

    6. **Mount in `express-app.ts`:** Import `portalAuthRoutes` and mount at `/api/portal`: `app.use("/api/portal", portalAuthRoutes);` — add between the challenger routes and institution routes lines.

    Run migration using the project pattern:
